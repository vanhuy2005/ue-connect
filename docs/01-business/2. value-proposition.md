# Value Proposition

## 1. Purpose

File này mô tả **value proposition** của UEConnect.

Mục tiêu:

- Làm rõ UEConnect tạo giá trị gì.
- Làm rõ giá trị đó dành cho ai.
- Làm rõ vì sao UEConnect khác các công cụ hiện có.
- Liên kết pain points với product value.
- Làm nền cho personas, business rules, feature priority, requirements và design direction.
- Giúp team ra quyết định khi có feature mới: feature đó có tạo thêm value thật không, hay chỉ làm app phình ra cho vui.

Value proposition không phải slogan đẹp. Nó là lời hứa sản phẩm có thể kiểm chứng bằng hành vi user, metric và trải nghiệm thật.

Nếu problem statement trả lời:

```txt
Vì sao UEConnect cần tồn tại?
````

thì value proposition trả lời:

```txt
UEConnect đem lại giá trị gì đủ khác biệt để user thật sự muốn dùng?
```

## 2. Core Value Proposition

```txt
UEConnect giúp sinh viên HCMUE kết nối đúng người, đúng cộng đồng và đúng sự hỗ trợ trong một môi trường social đã xác thực, an toàn và dành riêng cho HCMUE.
```

Nói ngắn hơn:

```txt
Kết nối và làm quen chuẩn HCMUEr.
```

Nói theo hướng product/business:

```txt
UEConnect tạo ra một social layer nội bộ cho HCMUE, kết hợp verified identity, student discovery, feed, messaging, mentor và community trong một nền tảng có trust, safety và governance rõ ràng.
```

## 3. Value Proposition Statement

## 3.1. Formal Statement

```txt
For HCMUE students who need to connect with peers, mentors, alumni and communities inside their university, UEConnect is a verified student social platform that helps them discover relevant UEers, start safe conversations, join communities and access mentorship through a trusted HCMUE-only environment.
```

## 3.2. Vietnamese Statement

```txt
Dành cho sinh viên HCMUE cần kết nối với bạn học, mentor, alumni và cộng đồng trong trường, UEConnect là nền tảng social đã xác thực giúp sinh viên khám phá UEers phù hợp, gửi lời chào, nhắn tin, tham gia CLB/cộng đồng và tiếp cận mentor trong một môi trường HCMUE-only đáng tin.
```

## 3.3. Short Statement

```txt
UEConnect là mạng xã hội nội bộ đáng tin cho sinh viên HCMUE, tập trung vào kết nối, học tập, mentor và cộng đồng.
```

## 3.4. Product Promise

```txt
UEConnect giúp mỗi sinh viên HCMUE tìm được đúng người để học cùng, đúng cộng đồng để tham gia và đúng mentor để hỏi trong hành trình đại học.
```

# 6. Value Pillar 01 — Verified Trust

## 6.1. Value Statement

```txt
UEConnect tạo niềm tin bằng verified identity: user biết rằng mình đang tương tác trong cộng đồng HCMUE đã được xác thực.
```

## 6.2. Why It Matters

Trong social platform nội bộ trường học, trust quan trọng hơn growth.

Nếu user không tin người bên kia là UEer thật:

- Discovery mất giá trị.
- Messaging dễ thành spam.
- Mentor request thiếu an toàn.
- Community dễ bị giả mạo.
- Admin khó kiểm soát.
- Nhà trường khó chấp nhận triển khai.

## 6.3. Product Capabilities

UEConnect tạo verified trust bằng:

- MSSV uniqueness.
- Upload minh chứng.
- Email `hcmue.edu.vn` nếu có.
- Admin verification review.
- Verified UEer badge.
- Primary role rõ.
- Permission phụ do admin cấp.
- Audit verification actions.

## 6.4. User Benefit

Student nhận được:

- Cảm giác an toàn hơn khi tương tác.
- Biết người khác thuộc HCMUE.
- Ít rủi ro gặp tài khoản giả.
- Có môi trường social đáng tin hơn public social.

Admin nhận được:

- Verification queue.
- Evidence review.
- Duplicate MSSV warning.
- Audit trail.

Institution nhận được:

- Trust model rõ.
- Governance rõ.
- Có cơ chế kiểm soát rủi ro.

## 6.5. Value Metric

| Metric                       | Meaning                           |
| ---------------------------- | --------------------------------- |
| Số tài khoản verified        | Trust adoption                    |
| Verification submission rate | User chấp nhận quy trình xác thực |
| Verification approval rate   | Flow/evidence rõ                  |
| Duplicate MSSV detection     | Trust rule hoạt động              |
| Average review time          | Khả năng vận hành admin           |

# 7. Value Pillar 02 — HCMUE-only Belonging

## 7.1. Value Statement

```txt
UEConnect tạo cảm giác thuộc về cộng đồng HCMUE bằng một không gian social chỉ dành cho HCMUE.
```

## 7.2. Why It Matters

Sinh viên không chỉ cần một mạng xã hội nữa. Họ cần một nơi có ngữ cảnh:

- Cùng trường.
- Cùng khoa.
- Cùng ngành.
- Cùng khóa.
- Cùng môn.
- Cùng CLB.
- Cùng trải nghiệm sinh viên.

Facebook, Threads, Instagram quá rộng. Zalo/Messenger quá đóng theo nhóm nhỏ. Portal trường quá hành chính. UEConnect nằm ở giữa: social, nhưng HCMUE-rooted.

## 7.3. Product Capabilities

- HCMUE-only positioning.
- UEer identity.
- Verified UEer badge.
- Profile có khoa/ngành/cohort.
- Discovery theo context HCMUE.
- Community/CLB HCMUE.
- Mentor/alumni/advisor gắn với HCMUE.

## 7.4. User Benefit

Student cảm thấy:

- Mình thuộc về một cộng đồng cụ thể.
- Nội dung liên quan hơn.
- Người mình gặp có cùng bối cảnh.
- Việc làm quen tự nhiên hơn.
- Việc hỏi mentor/alumni ít ngại hơn.

## 7.5. Value Metric

| Metric                      | Meaning                      |
| --------------------------- | ---------------------------- |
| Profile completion rate     | User thể hiện identity HCMUE |
| Daily active verified users | Belonging/usage              |
| Community join count        | Community belonging          |
| Discovery profile view rate | Interest in UEers            |
| First-year activation rate  | Onboarding/belonging value   |

# 8. Value Pillar 03 — Contextual Connection

## 8.1. Value Statement

```txt
UEConnect giúp sinh viên kết nối đúng người dựa trên ngữ cảnh học tập, khoa, ngành, cohort, interest, mentor need và community.
```

## 8.2. Why It Matters

Kết nối ngẫu nhiên không đủ giá trị.

User cần biết:

```txt
Vì sao mình thấy người này?
Mình có điểm chung gì với họ?
Mình nên bắt đầu nói chuyện như thế nào?
```

Nếu không có context, discovery dễ trở thành lướt profile vô nghĩa hoặc tệ hơn, trượt về dating vibe. Nhân loại đã có quá đủ app lướt người rồi, cảm ơn.

## 8.3. Product Capabilities

- Discovery UEers.
- Discovery filters.
- Shared context display.
- Greeting.
- Optional greeting message.
- Connection states.
- Messaging permission sau accepted greeting.
- Save profile.
- Report/block.

## 8.4. User Benefit

Student có thể:

- Tìm bạn cùng khoa.
- Tìm bạn cùng môn.
- Tìm người cùng interest.
- Gửi lời chào dễ hơn.
- Bắt đầu conversation có lý do.
- Tránh inbox ngẫu nhiên, thiếu ngữ cảnh.

## 8.5. Value Metric

| Metric                               | Meaning                       |
| ------------------------------------ | ----------------------------- |
| Số lời chào được gửi                 | Discovery/connection activity |
| Greeting acceptance rate             | Connection relevance          |
| Conversations started after greeting | Connection value              |
| Filter usage                         | Contextual search need        |
| Block/report rate in discovery       | Safety/quality signal         |

# 10. Value Pillar 05 — Mentor & Growth Support


- Có ngữ cảnh.
- Có request rõ.
- Có availability.
- Có accept/decline.
- Có messaging sau khi được chấp nhận.
- Có safety/report.

## 10.3. Product Capabilities

- Mentor list.
- Mentor profile.
- Support topics.
- Mentor request.
- Request status.
- Mentor availability.
- Accepted request opens conversation.
- Alumni/advisor verification.
- Report mentor interaction.

## 10.4. User Benefit

Student có thể:

- Tìm người phù hợp để hỏi.
- Gửi request có chủ đề rõ.
- Đỡ ngại inbox trực tiếp.
- Nhận hỗ trợ học tập/career/community.
- Học từ người đi trước.

Mentor/alumni/advisor có thể:

- Nhận request có context.
- Quản lý availability.
- Accept/decline.
- Không bị spam inbox tự do.
- Hỗ trợ sinh viên trong phạm vi phù hợp.

# 12. Value Pillar 07 — Safety, Privacy & Governance

## 12.1. Value Statement

```txt
UEConnect tạo môi trường social an toàn hơn bằng report, block, moderation, auto-hide, keyword flagging, privacy controls và audit.
```

## 12.2. Why It Matters

Social platform trong môi trường trường học cần kiểm soát rủi ro tốt hơn public social thông thường.

Các rủi ro chính:

- Giả mạo.
- Spam.
- Quấy rối.
- Dating/sexual content.
- Lộ thông tin cá nhân.
- Scam/lừa đảo.
- Nội dung vi phạm bản quyền.
- Ngôn từ công kích.
- Nội dung chính trị nhạy cảm.

## 12.3. Product Capabilities

- Report user/profile/post/comment/message/community.
- Block user.
- Moderation queue.
- Admin action.
- Auto-hide nếu nhiều report.
- Keyword flagging.
- Admin reason.
- Application audit log.
- SQL Server Audit direction.
- Evidence privacy.
- Privacy settings.

## 12.4. User Benefit

Student nhận được:

- Quyền tự bảo vệ bằng report/block.
- Ít rủi ro spam/quấy rối hơn.
- Cảm giác nền tảng có quản lý.
- Trust cao hơn khi tương tác.

Admin nhận được:

- Công cụ xử lý vi phạm.
- Queue rõ.
- Action rõ.
- Audit rõ.

Institution nhận được:

- Governance tốt hơn.
- Có khả năng truy vết.
- Giảm rủi ro triển khai social product.

## 12.5. Value Metric

| Metric                 | Meaning                 |
| ---------------------- | ----------------------- |
| Report resolution time | Safety operation health |
| Auto-hide review time  | Risk response           |
| Block rate             | Abuse/friction signal   |
| Repeat offender count  | Policy enforcement      |
| Audit coverage rate    | Governance maturity     |
