# User Journey

## 1. Purpose

File này mô tả **user journey** của các nhóm người dùng chính trong UEConnect.

Mục tiêu:

- Mô tả hành trình của từng persona khi tương tác với UEConnect.
- Làm rõ các giai đoạn: awareness, onboarding, verification, activation, engagement, retention, safety và support.
- Chỉ ra user goal, pain point, touchpoint, system response và opportunity ở từng bước.
- Làm nền cho user flow, use case, page specs, functional requirements và acceptance criteria.
- Đảm bảo journey bám đúng product vision: HCMUE-only, verified-first, student-first, social nhưng không dating, safety-first và enterprise-ready.

User journey không phải flowchart trang trí. Nó là cách ta nhìn sản phẩm qua mắt người dùng trước khi ép họ bấm qua 17 màn hình mà dev gọi là “hợp lý”. Người dùng không quan tâm database bạn đẹp thế nào. Tàn nhẫn, nhưng đúng.

---

## 2. Journey Design Principles

Tất cả journey của UEConnect phải bám các nguyên tắc:

```txt
01. Trust before full access
02. Verification must explain value
03. Profile setup must be progressive
04. Discovery must be contextual, not dating
05. Messaging must be permission-based
06. Mentor request must have clear context
07. Community must have governance
08. Safety actions must be reachable
09. Admin actions must be auditable
10. Mobile journey must be app-like, not desktop-shrunk
````

# 5. Global User Journey Model

## 5.1. High-level Lifecycle

```txt
Awareness
→ Registration
→ Verification
→ Account Status
→ Profile Setup
→ Activation
→ Core Engagement
→ Connection / Mentor / Community
→ Retention
→ Safety / Support
```

## 5.2. Access Gates

UEConnect không mở full product ngay sau registration.

```txt
Guest
  → Can view landing, register, login

Registered but unverified
  → Can submit verification, view account status, resubmit evidence

Verified UEer
  → Can access feed, profile, discovery, greeting, mentor, community, messaging with permission

Restricted / Suspended / Banned
  → Access limited based on account status
```

## 5.3. Core Product Loop

```txt
Verified UEer
  → Complete profile
  → Discover relevant people/content/community
  → Interact through feed/greeting/mentor/community
  → Receive notifications
  → Return to continue conversations and activity
```

## 5.4. Trust & Safety Loop

```txt
User sees risky content/behavior
  → Report or block
  → System creates report
  → Admin reviews
  → Admin takes action
  → Action is audited
  → User/content status updates if needed
```

# 6. Journey 01 — Guest to Registered User

## 6.1. Primary Persona

```txt
GS-01 — Guest đang tìm hiểu UEConnect
```

## 6.2. Journey Goal

Guest cần hiểu:

```txt
UEConnect là gì?
Có dành cho mình không?
Vì sao cần verified?
Có phải dating app không?
Đăng ký như thế nào?
```

## 6.3. Journey Stages

| Stage       | User Goal                | Touchpoint                            | Pain Point                  | UEConnect Response                           |
| ----------- | ------------------------ | ------------------------------------- | --------------------------- | -------------------------------------------- |
| Discover    | Biết UEConnect tồn tại   | Link landing, bạn bè, CLB, giới thiệu | Không biết app là gì        | Landing giải thích ngắn gọn                  |
| Understand  | Hiểu value               | Landing hero, feature blocks          | Sợ app mơ hồ hoặc dating    | Copy rõ: HCMUE-only, verified, student-first |
| Trust Check | Biết vì sao cần xác thực | Verification explanation              | Ngại upload minh chứng      | Privacy note, evidence reason                |
| Decide      | Có nên đăng ký không     | CTA register                          | Không rõ điều kiện tham gia | Nêu rõ dành cho HCMUE                        |
| Register    | Tạo tài khoản            | Signup form                           | Form dài, rối               | Form tối giản, step rõ                       |

## 6.4. Happy Path

```txt
Guest mở landing page
→ Đọc product promise
→ Hiểu UEConnect dành cho HCMUE
→ Click Đăng ký
→ Nhập thông tin cơ bản
→ Tạo account thành công
→ Được chuyển đến Verification Journey
```

## 6.5. Failure / Edge Paths

| Edge Case                              | Expected Handling                                              |
| -------------------------------------- | -------------------------------------------------------------- |
| Guest hiểu nhầm là dating app          | Landing/copy không dùng match, swipe, crush, hot               |
| Guest không biết vì sao cần minh chứng | Hiển thị explanation trước verification                        |
| Guest không thuộc HCMUE                | Không cho submit full verification hoặc giải thích eligibility |
| Signup lỗi                             | Hiển thị error cụ thể                                          |
| Email đã tồn tại                       | Gợi ý login/reset password                                     |

## 6.6. Required Pages

```txt
Landing Page
Register Page
Login Page
Verification Explanation Section
Privacy / Community Guidelines Link
```

## 6.7. Success Metrics

```txt
Landing → Register conversion
Register completion rate
Verification start rate
Drop-off before verification
```

# 7. Journey 02 — Registered User to Verified UEer

## 7.1. Primary Personas

```txt
UV-01 — User pending verification
ST-01 / ST-02 / ST-03 / ST-04 — Student
AL-01 / AL-02 — Alumni
AD-01 — Advisor
```

## 7.2. Journey Goal

User cần submit verification và hiểu trạng thái xét duyệt.

UEConnect cần xác thực:

```txt
User thuộc HCMUE hoặc có liên hệ hợp lệ với HCMUE.
MSSV không bị trùng.
Role ban đầu phù hợp.
Evidence được lưu và xử lý an toàn.
```

## 7.3. Journey Stages

| Stage              | User Goal                             | Touchpoint              | Pain Point                  | UEConnect Response             |
| ------------------ | ------------------------------------- | ----------------------- | --------------------------- | ------------------------------ |
| Start verification | Biết cần làm gì                       | Verification onboarding | Sợ quy trình dài            | Chia step rõ                   |
| Select identity    | Chọn Student/Alumni/Advisor           | Role selection          | Không biết chọn gì          | Giải thích từng role           |
| Enter information  | Nhập MSSV/email/thông tin             | Verification form       | Nhập sai, không biết format | Validation rõ                  |
| Upload evidence    | Tải minh chứng                        | Evidence upload         | Lo privacy                  | Privacy note, allowed evidence |
| Submit             | Gửi xét duyệt                         | Review screen           | Sợ thiếu thông tin          | Summary trước submit           |
| Wait               | Theo dõi trạng thái                   | Account status page     | Pending mơ hồ               | Status rõ                      |
| Review outcome     | Biết approved/rejected/need more info | Notification/status     | Không biết sửa gì           | Reason + resubmit path         |

## 7.4. Happy Path — Student

```txt
Registered user chọn Student
→ Nhập MSSV
→ Hệ thống kiểm tra MSSV chưa bị dùng
→ Upload thẻ sinh viên / giấy nhập học / bảng điểm
→ Submit verification
→ Account chuyển Pending
→ Admin approve
→ User nhận notification
→ Account thành Verified UEer
→ User vào Profile Setup
```

## 7.5. Happy Path — Alumni

```txt
Registered user chọn Alumni
→ Nhập thông tin liên quan HCMUE
→ Upload bằng tốt nghiệp / minh chứng từng học tại HCMUE
→ Submit verification
→ Admin review
→ Admin approve
→ Account có primary role Alumni
→ User vào Alumni Profile Setup
```

## 7.6. Happy Path — Advisor

```txt
Registered user chọn Advisor
→ Nhập thông tin advisor/cố vấn
→ Upload minh chứng phù hợp hoặc chờ admin xác nhận
→ Submit verification
→ Admin review
→ Admin approve
→ Account có primary role Advisor
→ User vào Advisor Profile Setup
```

## 7.7. Edge Paths

| Edge Case                        | Expected Handling                         |
| -------------------------------- | ----------------------------------------- |
| MSSV đã tồn tại                  | Chặn submit, hướng dẫn recovery/support   |
| Evidence thiếu/mờ                | Admin chọn Need more information          |
| User chọn sai role               | Admin reject/request correction           |
| User không có email hcmue.edu.vn | Cho dùng evidence khác                    |
| Alumni không nhớ MSSV            | Cho evidence alumni khác                  |
| Upload lỗi                       | Retry, giữ form state                     |
| Admin reject                     | Hiển thị reason, cho resubmit nếu phù hợp |
| Pending quá lâu                  | Hiển thị status + support path            |

## 7.8. Required Pages

```txt
Verification Role Selection
Student Verification Form
Alumni Verification Form
Advisor Verification Form
Evidence Upload
Verification Review Summary
Account Status Page
Verification Result Page / State
```

## 7.9. Required System States

```txt
draft
pending
approved
rejected
need_more_information
suspended
```

## 7.10. Success Metrics


# 8. Journey 03 — Verified Student Activation

## 8.1. Primary Personas

```txt
ST-01, ST-02, ST-03, ST-04, ST-05, ST-06
```

## 8.2. Journey Goal

Sau khi verified, student cần nhanh chóng setup profile và thấy value đầu tiên.

Value đầu tiên có thể là:

```txt
Thấy feed có nội dung liên quan

Gửi lời chào đầu tiên
Tham gia community/CLB
Tìm mentor phù hợp
```

## 8.3. Journey Stages

| Stage         | User Goal                        | Touchpoint               | Pain Point                | UEConnect Response       |
| ------------- | -------------------------------- | ------------------------ | ------------------------- | ------------------------ |
| Enter product | Biết làm gì tiếp                 | Welcome screen           | App trống/rối             | Activation checklist     |
| Setup profile | Tạo identity                     | Profile setup            | Form dài                  | Progressive setup        |
| Personalize   | Chọn khoa/ngành/interests        | Interest step            | Không muốn điền quá nhiều | Required + optional rõ   |
| First value   | Tìm người/nội dung/cộng đồng     | Feed/discovery/community | Không biết bắt đầu đâu    | Suggestions              |
| First action  | Gửi lời chào/post/join community | CTA                      | Ngại tương tác            | CTA nhẹ, copy thân thiện |
| Return        | Nhận notification                | Notification center      | Không có lý do quay lại   | Activity notifications   |

## 8.4. Happy Path

```txt
Verified student đăng nhập
→ Thấy welcome + activation checklist
→ Upload avatar hoặc skip
→ Nhập display name, khoa, ngành, cohort
→ Chọn interests và learning goals

→ Thấy suggested UEers/community/mentor
→ Gửi lời chào hoặc join community
```

## 8.5. Activation Checklist


# 10. Journey 05 — Peer Discovery & Greeting

## 10.1. Primary Personas

```txt
ST-01, ST-02, ST-05
```

## 10.2. Journey Goal

User muốn tìm UEers phù hợp và bắt đầu kết nối mà không bị dating vibe.

## 10.3. Journey Stages

| Stage    | User Goal               | Touchpoint          | Pain Point                   | UEConnect Response         |
| -------- | ----------------------- | ------------------- | ---------------------------- | -------------------------- |
| Browse   | Xem UEers gợi ý         | Discovery page      | Không biết vì sao được gợi ý | Shared context             |
| Evaluate | Xem profile card        | Card/profile detail | Thiếu thông tin              | Khoa, ngành, interest, bio |
| Start    | Mở lời                  | Gửi lời chào        | Ngại inbox trực tiếp         | Greeting nhẹ               |
| Wait     | Chờ phản hồi            | Greeting status     | Không biết đã gửi chưa       | Pending state              |
| Respond  | Accept/decline lời chào | Notification        | Sợ bị ép kết nối             | Accept/decline rõ          |
| Connect  | Bắt đầu conversation    | Messaging unlock    | Spam risk                    | Permission-based messaging |

## 10.4. Happy Path

```txt
Verified student mở Discovery
→ Xem card UEer cùng khoa/cùng interest
→ Mở profile detail
→ Click Gửi lời chào
→ Viết message ngắn hoặc dùng suggested greeting
→ Receiver nhận notification
→ Receiver accept
→ Connection được tạo
→ Conversation được mở
```

## 10.5. Greeting Copy Examples

Good:

```txt
Chào bạn, mình thấy tụi mình cùng khoa nên muốn kết nối.
Chào bạn, mình cũng đang quan tâm chủ đề này.
Chào bạn, mình muốn hỏi thêm về môn này nếu bạn tiện.
```

Do not use:

```txt
Match với mình nhé.
Crush bạn rồi.
Swipe phải.
Bạn hot quá.
```

Vâng, cấm rõ vậy vì chỉ cần một câu sai là product từ “student-first” thành “sao app trường giống Tinder vậy?”.

## 10.6. Edge Paths

| Edge Case                     | Expected Handling                                  |
| ----------------------------- | -------------------------------------------------- |
| User đã gửi greeting          | Disable CTA, show pending                          |
| Receiver decline              | Sender thấy declined hoặc không thông báo quá nặng |
| User block sender             | Chặn tương tác tiếp                                |
| Sender spam greeting          | Rate limit                                         |
| Profile bị report             | Tạo report item                                    |
| Discovery không có kết quả    | Empty state + chỉnh filter                         |
| User tắt discovery visibility | Không xuất hiện trong discovery                    |

## 10.7. Required Pages

```txt
Discovery Page
Discovery Filter
Profile Card
Profile Detail
Greeting Modal
Greeting Status
Notification Center
Conversation Page
```

## 10.8. Success Metrics

```txt
Discovery profile views
Greetings sent
Greeting acceptance rate
Conversation started after greeting
Report/block rate from discovery
Filter usage
```

# 11. Journey 06 — Social Feed Engagement

```txt
ST-02, ST-03, ST-06, AL-01
```

## 11.2. Journey Goal

User muốn đọc, đăng bài, hỏi đáp, comment và tương tác trong môi trường HCMUE.

## 11.3. Journey Stages

| Stage       | User Goal              | Touchpoint      | Pain Point              | UEConnect Response            |
| ----------- | ---------------------- | --------------- | ----------------------- | ----------------------------- |
| Browse feed | Xem nội dung liên quan | Home feed       | Feed rác/trống          | Seed content, community posts |
| Create      | Đăng bài               | Composer        | Không biết viết gì      | Prompt/gợi ý                  |
| Discuss     | Comment/reply          | Comment section | Sợ public quá           | Tone thân thiện               |
| Save        | Lưu bài hữu ích        | Save action     | Không tìm lại được      | Saved posts                   |
| Report      | Báo cáo nội dung xấu   | Report menu     | Không biết report ở đâu | Report accessible             |
| Return      | Theo dõi phản hồi      | Notification    | Không biết ai tương tác | Notification                  |

## 11.4. Happy Path

```txt
Verified user mở Home Feed
→ Đọc bài liên quan HCMUE

```

## 11.5. Post Types

```txt
Question post
Experience sharing
Learning post
Community post
Mentor/alumni insight
Club announcement
```

## 11.6. Edge Paths

| Edge Case            | Expected Handling                   |
