# Goals and Success Metrics

## 1. Purpose

File này định nghĩa **business goals**, **product goals** và **success metrics** của UEConnect.

Mục tiêu:

- Làm rõ UEConnect cần đạt được điều gì.
- Liên kết goal với problem, value proposition, personas và roadmap.
- Xác định metric nào là primary, secondary, operational và guardrail.
- Làm rõ metric nào đo product value, metric nào đo trust, metric nào đo safety, metric nào đo vận hành.
- Làm nền cho KPI tree, analytics event, dashboard, acceptance criteria và product decision.
- Tránh việc đánh giá sản phẩm bằng cảm giác.

Một goal tốt phải trả lời:

```txt
Ta muốn thay đổi điều gì?
Cho ai?
Vì sao điều đó quan trọng?
Đo bằng gì?
````

Một metric tốt phải trả lời:

```txt
Dữ liệu này nói gì về sức khỏe thật của sản phẩm?
Nó có thể hành động được không?
Nó có bị vanity không?
```

Metric không tồn tại để làm dashboard trông thông minh. Dashboard đẹp mà không giúp ra quyết định thì chỉ là tranh treo tường có số.

## 2. Measurement Philosophy

UEConnect không đo thành công bằng số lượng feature đã làm.

UEConnect đo thành công bằng việc:

```txt
Sinh viên HCMUE có thật sự được xác thực không?
Sinh viên có thật sự đăng bài không?
Sinh viên có thật sự gửi lời chào không?
Sinh viên có thật sự tìm mentor không?
Sinh viên có thật sự tham gia community không?
Admin có thật sự vận hành verification/moderation được không?
Hệ thống có thật sự an toàn và đáng tin hơn không?
```

## 2.1. Product Measurement Principles

UEConnect dùng các nguyên tắc đo lường sau:

```txt
01. Measure behavior, not opinions only
02. Measure trust before growth
03. Measure quality, not only quantity
04. Measure safety as a core success factor
05. Measure admin operation, not only user activity
06. Measure cohort differences
07. Measure funnel drop-off
08. Measure retention after activation
09. Avoid vanity metrics
10. Every key metric must map to a product decision
```

## 2.2. Primary Measurement Focus

Trong MVP, UEConnect ưu tiên 4 chỉ số chính:

```txt
Số tài khoản verified
Số bài viết mỗi ngày
Số mentor request
Số lời chào được gửi
```

4 metric này được chọn vì chúng đại diện cho 4 trụ cột giá trị:

| Metric                | Product Value Measured       |
| --------------------- | ---------------------------- |
| Số tài khoản verified | Trust / adoption             |
| Số bài viết mỗi ngày  | Social feed activity         |
| Số mentor request     | Mentor / growth support      |
| Số lời chào được gửi  | Discovery / connection value |

Nếu 4 metric này không chạy, sản phẩm có đẹp đến đâu cũng chỉ là UI có tham vọng. Một loại sinh vật Figma rất phổ biến.

# 4. North Star Goal

## 4.1. North Star Goal Statement

```txt
UEConnect giúp sinh viên HCMUE kết nối đúng người, đúng cộng đồng và đúng sự hỗ trợ trong một môi trường verified, an toàn và có governance.
```

## 4.2. North Star Metric Candidate

North Star Metric chính nên là:

```txt
Weekly Meaningful Connections
```

## 4.3. Definition — Weekly Meaningful Connections

Một **Meaningful Connection** có thể được tính khi có ít nhất một trong các event sau:

```txt
Greeting accepted
Conversation started after accepted greeting
Mentor request accepted
Community joined
Meaningful comment interaction on post
```

## 4.4. Why This Metric

Metric này tốt hơn chỉ đếm login hoặc page view vì nó đo hành vi có giá trị thật:

- User không chỉ vào app.
- User không chỉ lướt.
- User có kết nối.
- User có tham gia.
- User có nhận hỗ trợ.
- User có tương tác có ngữ cảnh.

## 4.5. MVP Practical Version

Trong MVP, nếu chưa đủ tracking phức tạp, có thể dùng metric đơn giản hơn:

```txt
Weekly Active Verified UEers who perform at least one meaningful action
```

Meaningful actions gồm:

```txt
Create post
Comment
Send greeting
Accept greeting
Send mentor request
Accept mentor request
Join community
Send message after connection
```

## 4.6. Guardrail

Không được dùng metric vanity như:

```txt
Total registered accounts
Total page views
Total impressions
Total profile views without action
```

Các metric này có thể theo dõi, nhưng không nên là North Star. Vì page view cao có thể chỉ chứng minh người dùng đang lạc trong UI. Chúc mừng, nhưng không hẳn.

# 5. Goal G-01 — Trust & Verification

## 5.1. Goal Statement

```txt
UEConnect phải xây dựng được một cộng đồng HCMUE có xác thực, nơi user đủ tin tưởng để tạo profile, tương tác, gửi lời chào, tham gia community và tìm mentor.
```

## 5.2. Why It Matters

Verified identity là nền tảng của UEConnect.

Không có trust:

- Discovery mất ý nghĩa.
- Messaging dễ bị spam.
- Mentor không muốn tham gia.
- Community dễ giả mạo.
- Admin không thể vận hành.
- Nhà trường khó chấp nhận.

## 5.3. Key Metrics

| Metric                           | Definition                                 | Type        | Priority |
| -------------------------------- | ------------------------------------------ | ----------- | -------: |
| Verified Accounts                | Số tài khoản đã được approved verification | Primary     |       P0 |
| Verification Submission Rate     | % registered users submit verification     | Funnel      |       P0 |
| Verification Approval Rate       | % verification requests được approve       | Quality     |       P0 |
| Verification Rejection Rate      | % verification requests bị reject          | Quality     |       P1 |
| Need More Information Rate       | % request cần bổ sung                      | Quality/Ops |       P1 |
| Average Verification Review Time | Thời gian trung bình admin xử lý request   | Operational |       P0 |
| Duplicate MSSV Detection Count   | Số case MSSV trùng được phát hiện          | Trust       |       P0 |
| Evidence Upload Drop-off Rate    | % user bỏ ở bước upload evidence           | UX          |       P0 |

## 5.4. Metric Formula

```txt
Verification Submission Rate =
Number of users who submit verification / Number of registered users
```

```txt
Verification Approval Rate =
Number of approved verification requests / Number of submitted verification requests
```

```txt
Average Verification Review Time =
Average(verification_decision_time - verification_submitted_time)
```

```txt
Evidence Upload Drop-off Rate =
Users who reach evidence upload but do not submit / Users who reach evidence upload
```

## 5.5. Success Indicators

MVP được xem là có trust foundation tốt khi:

- User hiểu vì sao cần verification.
- Tỷ lệ submit verification không quá thấp.
- Admin xử lý verification được.
- User bị need_more_information có thể submit lại.
- Duplicate MSSV được phát hiện.
- Evidence được xử lý như dữ liệu nhạy cảm.
- Approved user có thể vào profile setup.

## 5.6. Product Decisions Triggered by Metrics

| Signal                         | Possible Decision                                                   |
| ------------------------------ | ------------------------------------------------------------------- |
| Evidence Upload Drop-off cao   | Cải thiện copy privacy, giảm friction, thêm evidence examples       |
| Need More Information Rate cao | Làm rõ guideline upload evidence                                    |
| Review Time cao                | Cải thiện admin queue/filter                                        |
| Duplicate MSSV cao             | Tăng support/recovery flow                                          |
| Approval Rate quá thấp         | Kiểm tra rule verification có quá nghiêm hoặc user submit sai không |
| Submission Rate thấp           | Landing/onboarding chưa giải thích đủ value                         |

# 6. Goal G-02 — Student Activation

## 6.1. Goal Statement

```txt
Verified students phải nhanh chóng hiểu UEConnect dùng để làm gì và thực hiện được hành động có giá trị đầu tiên sau khi được duyệt.
```

## 6.2. Why It Matters

Verification chỉ là cửa vào. Nếu verified xong mà user không biết làm gì tiếp, product vẫn chết rất lịch sự.

Activation là giai đoạn chứng minh value đầu tiên.

## 6.3. Activation Definition

Một user được xem là **activated** khi họ hoàn thành ít nhất 2 trong các hành động sau trong 7 ngày đầu sau khi được approved:

```txt
Complete basic profile
Send first greeting
Join/follow a community
Create first post
Comment on a post
Send mentor request
Save a post/profile
Start first conversation
```

## 6.4. Key Metrics

| Metric                          | Definition                                            | Type       | Priority |
| ------------------------------- | ----------------------------------------------------- | ---------- | -------: |
| Profile Completion Rate         | % verified users hoàn thành profile cơ bản            | Activation |       P0 |
| First Meaningful Action Rate    | % verified users thực hiện action có giá trị đầu tiên | Activation |       P0 |
| Time to First Meaningful Action | Thời gian từ approved đến action đầu tiên             | Activation |       P0 |
| First Greeting Sent Rate        | % verified users gửi lời chào đầu tiên                | Activation |       P0 |
| First Community Join Rate       | % verified users join/follow community đầu tiên       | Activation |       P1 |
| First Mentor Request Rate       | % verified users gửi mentor request đầu tiên          | Activation |       P1 |
| Day-1 Return Rate               | % user quay lại sau ngày đầu approved                 | Retention  |       P1 |
| 7-day Activation Rate           | % user activated trong 7 ngày đầu                     | Activation |       P0 |

## 6.5. Metric Formula

```txt
Profile Completion Rate =
Users with completed basic profile / Approved users
```

```txt
First Meaningful Action Rate =
Approved users with at least one meaningful action / Approved users
```

```txt
7-day Activation Rate =
Users who complete activation definition within 7 days / Newly approved users
```

## 6.6. Success Indicators

Activation tốt khi:

- User hoàn thiện profile cơ bản.
- User thấy gợi ý UEers/community/mentor.
- User gửi lời chào hoặc tham gia community.
- User quay lại sau khi nhận notification.
- User không bị kẹt ở màn welcome.

## 6.7. Product Decisions Triggered by Metrics

| Signal                    | Possible Decision                                 |
| ------------------------- | ------------------------------------------------- |
| Profile Completion thấp   | Giảm required fields, thêm progressive completion |
| Time to First Action cao  | Cải thiện onboarding checklist                    |
| First Greeting thấp       | Discovery chưa hấp dẫn hoặc CTA chưa rõ           |
| First Community Join thấp | Community listing/gợi ý chưa tốt                  |
| First Mentor Request thấp | Mentor value chưa rõ hoặc form quá nặng           |
| Day-1 Return thấp         | Notification/content chưa tạo lý do quay lại      |

# 7. Goal G-03 — Social Engagement

## 7.1. Goal Statement

```txt
UEConnect phải tạo được social feed có hoạt động thật, nơi sinh viên có thể đăng bài, hỏi đáp, comment và tương tác trong phạm vi HCMUE.
```

## 7.2. Why It Matters

Feed là thói quen sử dụng hằng ngày.

Nếu không có social engagement:

- User không quay lại.
- Discovery thiếu ngữ cảnh.
- Community ít hoạt động.
- Mentor/alumni content không lan tỏa.
- Product trở thành danh bạ có chat.

Một danh bạ có chat thì cũng hữu ích, nhưng không cần gọi là social platform cho hoành tráng.

## 7.3. Key Metrics

| Metric               | Definition                                        | Type                   | Priority |
| -------------------- | ------------------------------------------------- | ---------------------- | -------: |
| Daily Posts          | Số bài viết mỗi ngày                              | Primary                |       P0 |
| Active Posters       | Số user đăng ít nhất 1 bài trong khoảng thời gian | Engagement             |       P0 |
| Comments per Post    | Số comment trung bình trên mỗi post               | Engagement Quality     |       P1 |
| Commenting Users     | Số user comment                                   | Engagement             |       P1 |
| Save Rate            | % post được save                                  | Content Value          |       P1 |
| Like Rate            | % post có like                                    | Lightweight Engagement |       P2 |
| Feed Return Rate     | % user quay lại feed sau lần đầu                  | Retention              |       P1 |
| Report Rate per Post | Số report/post                                    | Safety Guardrail       |       P0 |
| Hidden Content Rate  | % content bị hide/moderate                        | Safety Guardrail       |       P1 |

## 7.4. Metric Formula

```txt
Daily Posts =
Count(posts created per day)
```

```txt
Comments per Post =
Total comments / Total posts
```

```txt
Save Rate =
Saved posts / Viewed posts
```

```txt
Report Rate per Post =
Reports targeting posts / Total posts
```

## 7.5. Success Indicators

Feed thành công khi:

- Có bài viết mới mỗi ngày.
- User không chỉ đọc mà còn comment.
- Nội dung có liên quan đến student/HCMUE context.
- Bài hữu ích được save.
- Report rate không vượt ngưỡng rủi ro.
- Feed không biến thành spam hoặc portal thông báo khô khan.

## 7.6. Product Decisions Triggered by Metrics

| Signal                 | Possible Decision                          |
| ---------------------- | ------------------------------------------ |
| Daily Posts thấp       | Thêm prompt, seed content, community posts |
| Comments per Post thấp | Cải thiện post prompts, question posts     |
| Save Rate cao          | Phát triển saved posts/resource direction  |
| Report Rate cao        | Tăng moderation, policy reminders          |
| Feed Return thấp       | Cải thiện notification/content relevance   |
| Active Posters thấp    | Giảm friction composer, gợi ý topic        |

# 8. Goal G-04 — Discovery & Connection

## 8.1. Goal Statement

```txt
UEConnect phải giúp sinh viên khám phá UEers phù hợp và bắt đầu kết nối có ngữ cảnh mà không tạo dating vibe.
```

## 8.2. Why It Matters

Discovery là điểm khác biệt lớn của UEConnect.

Nhưng discovery cũng là vùng rủi ro cao vì dễ bị hiểu sai thành dating nếu dùng sai copy, icon, interaction hoặc visual. Con người chỉ cần thấy card profile + thao tác lướt là bắt đầu suy diễn. Thật phiền, nhưng phải thiết kế để tránh.

## 8.3. Key Metrics

| Metric                               | Definition                                    | Type             | Priority |
| ------------------------------------ | --------------------------------------------- | ---------------- | -------: |
| Greetings Sent                       | Số lời chào được gửi                          | Primary          |       P0 |
| Greeting Acceptance Rate             | % lời chào được accept                        | Quality          |       P0 |
| Discovery Profile Views              | Số lượt xem profile từ Discovery              | Engagement       |       P1 |
| Discovery Filter Usage               | % sessions có dùng filter                     | Relevance        |       P1 |
| Conversations Started after Greeting | Số conversation bắt đầu sau accepted greeting | Value            |       P0 |
| Saved Profiles                       | Số profile được save                          | Intent           |       P1 |
| Greeting Decline Rate                | % greeting bị decline                         | Quality          |       P1 |
| Block Rate from Discovery            | % interaction dẫn đến block                   | Safety Guardrail |       P0 |
| Report Rate from Discovery           | % profile/greeting bị report                  | Safety Guardrail |       P0 |

## 8.4. Metric Formula

```txt
Greeting Acceptance Rate =
Accepted greetings / Sent greetings
```

```txt
Conversation Started after Greeting =
Conversations created from accepted greetings
```

```txt
Block Rate from Discovery =
Blocks triggered from discovery context / Discovery interactions
```

## 8.5. Success Indicators

Discovery thành công khi:

- User gửi lời chào.
- Lời chào có tỷ lệ accept hợp lý.
- Accepted greeting dẫn đến conversation.
- User dùng filter hoặc click profile vì thấy context.
- Report/block rate không cao.
- UI/copy không bị hiểu là dating.

## 8.6. Product Decisions Triggered by Metrics

| Signal                                 | Possible Decision                              |
| -------------------------------------- | ---------------------------------------------- |
| Greetings Sent thấp                    | Cải thiện card, CTA, suggestions               |
| Acceptance Rate thấp                   | Gợi ý chưa relevant hoặc greeting copy kém     |
| Report/Block Rate cao                  | Có spam/abuse, cần rate limit hoặc privacy     |
| Filter Usage thấp                      | Filter khó dùng hoặc default suggestion đủ tốt |
| Profile Views cao nhưng Greetings thấp | Profile thiếu trust/CTA chưa rõ                |
| Conversations Started thấp             | Messaging unlock/copy chưa rõ                  |

# 9. Goal G-05 — Mentor & Growth Support

## 9.1. Goal Statement

```txt
UEConnect phải giúp sinh viên tiếp cận mentor, alumni hoặc advisor phù hợp thông qua request có cấu trúc, rõ ngữ cảnh và không gây áp lực kiểu LinkedIn.
```

## 9.2. Why It Matters

Mentor là một pillar chính của UEConnect.

Nếu mentor hoạt động tốt:

- Sinh viên có người hỏi.
- Alumni có cách đóng góp.
- Advisor có request rõ.
- Product có chiều sâu hơn social feed.
- HCMUE network có giá trị dài hạn hơn.

Nếu mentor hoạt động tệ:

- Request bị bỏ qua.
- Mentor bị spam.
- Student mất niềm tin.
- Product thành “có module mentor cho đẹp”.

## 9.3. Key Metrics

| Metric                            | Definition                                 | Type             | Priority |
| --------------------------------- | ------------------------------------------ | ---------------- | -------: |
| Mentor Requests                   | Số mentor request được gửi                 | Primary          |       P0 |
| Mentor Request Acceptance Rate    | % request được accept                      | Quality          |       P0 |
| Mentor Response Time              | Thời gian từ request đến phản hồi đầu tiên | Operational      |       P1 |
| Completed Mentor Interactions     | Số mentor request được mark completed      | Value            |       P1 |
| Active Mentors                    | Số mentor available hoặc nhận request      | Supply           |       P0 |
| Mentor Availability Rate          | % mentor đang available/limited            | Supply Health    |       P1 |
| Request Decline Rate              | % request bị decline                       | Quality          |       P1 |
| Need More Information Rate        | % request cần bổ sung thông tin            | Request Quality  |       P1 |
| Mentor Pause Rate                 | % mentor pause availability                | Overload Signal  |       P1 |
| Report Rate in Mentor Interaction | Report liên quan mentor                    | Safety Guardrail |       P0 |

## 9.4. Metric Formula

```txt
Mentor Request Acceptance Rate =
Accepted mentor requests / Submitted mentor requests
```

```txt
Mentor Response Time =
First mentor action time - Mentor request submitted time
```

```txt
Active Mentors =
Mentors with availability status available or limited
```

## 9.5. Success Indicators

Mentor thành công khi:

- Student gửi mentor request.
- Mentor profile có support topics rõ.
- Mentor accept request phù hợp.
- Request dẫn đến conversation.
- Mentor không bị quá tải.
- Report rate trong mentor interaction thấp.
- Student cảm thấy đây là hỗ trợ, không phải career pressure.

## 9.6. Product Decisions Triggered by Metrics

| Signal                | Possible Decision                        |
| --------------------- | ---------------------------------------- |
| Mentor Requests thấp  | Mentor page chưa rõ value, CTA yếu       |
| Acceptance Rate thấp  | Request quality kém hoặc mentor mismatch |
| Response Time cao     | Mentor unavailable hoặc notification yếu |
| Mentor Pause Rate cao | Mentor overload, cần request cap         |
| Need More Info cao    | Request form thiếu guidance              |
| Active Mentors thấp   | Cần alumni/advisor activation            |
| Report Rate cao       | Cần policy/safety review                 |

## 9.7. Related Personas


## 11.1. Goal Statement


## 11.3. Key Metrics

| Metric                        | Definition                           | Type              | Priority |
| ----------------------------- | ------------------------------------ | ----------------- | -------: |
| Reports Submitted             | Số report được gửi                   | Safety Signal     |       P0 |

| Report Action Rate            | % report dẫn đến action              | Quality           |       P1 |
| Dismissed Report Rate         | % report bị dismiss                  | Quality           |       P1 |
| Auto-hide Events              | Số nội dung bị auto-hide             | Safety Automation |       P1 |
| Auto-hide False Positive Rate | % auto-hide được restore             | Quality Guardrail |       P1 |
| Keyword Flags                 | Số nội dung bị keyword flag          | Safety Signal     |       P1 |
| Keyword Flag Action Rate      | % keyword flag dẫn đến action        | Quality           |       P1 |
| Block Count                   | Số block user                        | Safety Signal     |       P0 |

## 11.4. Metric Formula



- Auto-hide không bị lạm dụng.

# 19. Guardrail Metrics

Guardrail metrics giúp tránh việc tối ưu một chỉ số nhưng phá sản phẩm.

Ví dụ: tăng Greetings Sent bằng cách mở spam greeting tự do là một thành tích khá ngu ngốc, nhưng dashboard có thể vẫn vỗ tay nếu không có guardrail.

| Daily Posts       | Report rate per post, hidden content rate      |
| Mentor Requests   | Mentor pause rate, decline rate                |
| Community Joins   | Community report rate, inactive community rate |
| Verified Accounts | Verification quality, duplicate MSSV issues    |
| Messaging Volume  | Message report/block rate                      |

## 19.2. Safety Guardrails

| Metric                   | Risk if High               |
| ------------------------ | -------------------------- |
| Report Rate              | Content/user safety issue  |
| Block Rate               | Harassment/spam/friction   |
| Auto-hide False Positive | Automation too aggressive  |
| Repeat Offender Count    | Enforcement weak           |
| Moderation Backlog       | Admin operation overloaded |

## 19.3. UX Guardrails

# 24. Anti-metrics / Vanity Metrics

Không dùng các metric sau làm chỉ số thành công chính:

| Metric                    | Why Not Enough                                   |
| ------------------------- | ------------------------------------------------ |
| Total registered users    | Không nói user có verified hay active không      |
| Total page views          | Có thể do user lạc UI                            |
| Total profile views       | Không nói có connection không                    |
| Total notifications sent  | Có thể là spam                                   |
| Total communities created | Không nói community có active không              |
| Total messages            | Có thể là spam nếu thiếu guardrail               |
| Total likes               | Tương tác nhẹ, dễ vanity                         |
| Time spent                | Có thể là engagement, cũng có thể là UI khó dùng |

Metric tốt phải liên quan đến value thật:

```txt
Verified identity
Meaningful connection
Social contribution
Mentor support
Community participation
Safety resolution
```
