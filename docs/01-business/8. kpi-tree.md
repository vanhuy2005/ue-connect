# KPI Tree

## 1. Purpose

File này định nghĩa **KPI Tree** của UEConnect.

Mục tiêu:

- Phân rã North Star Metric thành các nhóm KPI cụ thể.
- Kết nối business goals với product metrics, operational metrics và guardrail metrics.
- Xác định metric nào là output, metric nào là input, metric nào là warning signal.
- Làm rõ các event cần tracking để tính KPI.
- Làm nền cho analytics events, dashboard, acceptance criteria và product decision.
- Tránh việc đo quá nhiều số nhưng không biết số nào quan trọng.

KPI Tree không phải danh sách metric càng dài càng tốt. Một cây KPI tốt phải chỉ ra **nguyên nhân**, **kết quả** và **rủi ro**. Nếu chỉ gom số vào một file rồi gọi là analytics, thì Excel cũng đang bật cười ở đâu đó.

## 2. Measurement Model

UEConnect dùng mô hình đo lường theo 5 tầng:

```txt
North Star Metric
→ Output Metrics
→ Input Metrics
→ Funnel / Cohort Metrics
→ Guardrail Metrics
→ Tracking Events
````

## 2.1. North Star Metric

Metric cao nhất đại diện cho giá trị cốt lõi của sản phẩm.

```txt
Weekly Meaningful Connections
```

## 2.2. Output Metrics

Kết quả chính mà product muốn tạo ra.

Ví dụ:

```txt
Verified Accounts
Daily Posts
Greetings Sent
Mentor Requests
Community Joins
```

## 2.3. Input Metrics

Các hành vi đầu vào thúc đẩy output metrics.

Ví dụ:

```txt
Profile completion
Discovery opened
Mentor profile viewed
Community detail viewed
Post composer opened
```

## 2.4. Funnel Metrics

Đo user đi qua từng bước có thành công không.

Ví dụ:

```txt
Register → Verification → Approved → Profile Completed
Discovery → Greeting Sent → Greeting Accepted → Conversation Started
```

## 2.5. Guardrail Metrics

Các chỉ số cảnh báo nếu tăng trưởng làm hỏng trust, safety, UX hoặc vận hành.

Ví dụ:

```txt
Report rate
Block rate
Verification drop-off
Moderation backlog
Auto-hide false positive rate
```

## 2.6. Tracking Events

Các event thô cần log để tính metric.

Ví dụ:

```txt
verification_submitted
profile_completed
greeting_sent
mentor_request_submitted
report_submitted
```

# 3. North Star KPI

## 3.1. North Star Goal

```txt
UEConnect giúp sinh viên HCMUE kết nối đúng người, đúng cộng đồng và đúng sự hỗ trợ trong một môi trường verified, an toàn và có governance.
```

## 3.2. North Star Metric

```txt
Weekly Meaningful Connections
```

## 3.3. Definition

Một **Weekly Meaningful Connection** được tính khi trong một tuần có ít nhất một meaningful action sau:

```txt
Greeting accepted
Conversation started after accepted greeting
Mentor request accepted
Community joined/followed
Meaningful comment interaction
```

## 3.4. MVP Simplified Definition

Ở MVP, có thể dùng định nghĩa dễ tracking hơn:

```txt
Weekly Active Verified UEers with at least one meaningful action
```

Meaningful action gồm:

```txt
Create post
Comment
Send greeting
Accept greeting
Start conversation
Submit mentor request
Accept mentor request
Join community
Create community post
```

## 3.5. Formula

```txt
Weekly Meaningful Connections =
Count of unique verified users who perform at least one meaningful action in a calendar week
```

Hoặc bản event-based:

```txt
Weekly Meaningful Connections =
Count(greeting_accepted)
+ Count(conversation_started_after_greeting)
+ Count(mentor_request_accepted)
+ Count(community_joined)
+ Count(meaningful_comment_interaction)
```

## 3.6. Why This Metric

Metric này đo đúng giá trị lõi hơn các metric vanity như page views hoặc registered accounts.

Nó trả lời:

```txt
User có thật sự tạo kết nối có ý nghĩa không?
User có thật sự tham gia social/community/mentor không?
User có quay lại vì value thật không?
```

## 3.7. Guardrails for North Star

Không được tăng North Star bằng cách phá safety.

| Growth Attempt                       | Risk                 | Guardrail                        |
| ------------------------------------ | -------------------- | -------------------------------- |
| Tăng greeting bằng spam              | User bị làm phiền    | Report/block rate from discovery |
| Tăng post bằng clickbait             | Feed rác             | Report rate per post             |
| Tăng mentor request bằng form quá dễ | Mentor bị overload   | Mentor decline/pause rate        |
| Tăng community joins bằng auto-join  | Không có intent thật | Community active rate            |
| Tăng message bằng mở inbox tự do     | Spam/quấy rối        | Message report/block rate        |

# 5. KPI Branch 01 — Trust & Verification

## 5.1. Branch Goal

```txt
Xây dựng nền tảng verified HCMUE community đủ tin cậy để user yên tâm tương tác.
```

## 5.2. KPI Relationship

```txt
Verified Accounts
│
├── Registration Completion Rate
├── Verification Start Rate
├── Evidence Upload Completion Rate
├── Verification Submission Rate
├── Verification Approval Rate
├── Need More Information Resubmission Rate
└── Average Verification Review Time
```

## 5.3. Output Metrics

| KPI                              | Definition                        | Priority |
| -------------------------------- | --------------------------------- | -------: |
| Verified Accounts                | Tổng số account đã được approved  |       P0 |
| Verification Approval Rate       | % request được approve            |       P0 |
| Average Verification Review Time | Thời gian trung bình admin duyệt  |       P0 |
| Duplicate MSSV Detection Count   | Số case MSSV trùng được phát hiện |       P0 |

## 5.4. Input Metrics

| KPI                      | Definition                  | Why It Matters            |
| ------------------------ | --------------------------- | ------------------------- |
| Register Starts          | User bắt đầu đăng ký        | Đo acquisition intent     |
| Register Completions     | User tạo account thành công | Đo auth friction          |
| Verification Starts      | User bắt đầu verification   | Đo trust gate entry       |
| Evidence Uploads         | User upload evidence        | Đo readiness              |
| Verification Submissions | User gửi request            | Đo completed trust action |
| Admin Reviews Completed  | Admin xử lý request         | Đo operation throughput   |

## 5.5. Funnel Metrics

```txt
Landing Visit
→ Register Start
→ Register Complete
→ Verification Start
→ Evidence Upload
→ Verification Submit
→ Admin Review
→ Approved
→ Profile Setup Start
```

| Funnel Step                          | Metric                   |
| ------------------------------------ | ------------------------ |
| Landing → Register                   | Register Conversion Rate |
| Register → Verification Start        | Verification Start Rate  |
| Verification Start → Evidence Upload | Evidence Upload Rate     |
| Evidence Upload → Submit             | Verification Submit Rate |
| Submit → Approved                    | Approval Rate            |
| Need More Information → Resubmit     | Resubmission Rate        |
| Approved → Profile Start             | Profile Setup Start Rate |

## 5.6. Guardrail Metrics

| Guardrail                     | Risk Detected                        |
| ----------------------------- | ------------------------------------ |
| Evidence Upload Drop-off Rate | User sợ privacy hoặc form khó        |
| Need More Information Rate    | Evidence guideline chưa rõ           |
| Rejection Rate                | User submit sai hoặc policy quá chặt |
| Verification Queue Backlog    | Admin quá tải                        |
| Duplicate MSSV Conflict Rate  | Identity conflict                    |
| Evidence Access Audit Missing | Privacy/governance risk              |

## 5.7. Core Events

```txt
user_registered
verification_started
verification_evidence_uploaded
verification_submitted
verification_approved
verification_rejected
verification_need_more_information
verification_resubmitted
duplicate_mssv_detected
```

## 5.8. Product Decisions

| KPI Signal                   | Decision                                     |
| ---------------------------- | -------------------------------------------- |
| Evidence Upload Drop-off cao | Viết lại privacy copy, thêm example evidence |
| Queue Backlog cao            | Thêm filter/admin reviewer                   |
| Need More Information cao    | Cải thiện instruction upload                 |
| Duplicate MSSV cao           | Thêm account recovery/support flow           |
| Approval Rate thấp           | Review policy hoặc input validation          |

# 6. KPI Branch 02 — Activation

## 6.1. Branch Goal

```txt
Giúp verified user nhanh chóng thấy value đầu tiên sau khi được duyệt.
```

## 6.2. KPI Relationship

```txt
7-day Activation Rate
│
├── Profile Completion Rate
├── First Meaningful Action Rate
├── Time to First Meaningful Action
├── First Greeting Sent Rate

├── First Mentor Request Rate
```

## 6.3. Output Metrics

| KPI                             | Definition                                | Priority |
| ------------------------------- | ----------------------------------------- | -------: |
| 7-day Activation Rate           | % user activated trong 7 ngày đầu         |       P0 |
| First Meaningful Action Rate    | % user làm action có giá trị đầu tiên     |       P0 |
| Time to First Meaningful Action | Thời gian từ approved đến action đầu tiên |       P0 |
| Day-1 Return Rate               | % user quay lại sau ngày đầu              |       P1 |

## 6.4. Activation Definition

User được xem là activated khi hoàn thành profile cơ bản và thực hiện ít nhất 1 meaningful action:

```txt
Send greeting
Create post
Comment
Join community
Submit mentor request
Start conversation
Save post/profile
```

## 6.5. Input Metrics

| KPI                          | Definition                   |
| ---------------------------- | ---------------------------- |
| Profile Setup Starts         | User bắt đầu setup profile   |
| Basic Profile Completed      | User hoàn thành field cơ bản |

| Discovery Suggestions Viewed | User xem gợi ý UEers         |
| Community Suggestions Viewed | User xem gợi ý community     |
| Mentor Suggestions Viewed    | User xem gợi ý mentor        |

## 6.6. Funnel Metrics

```txt

→ Profile Setup Started
→ Basic Profile Completed
→ Suggestions Viewed
```

## 6.7. Guardrail Metrics

| Suggestion Click-through thấp | Gợi ý không relevant        |
| Day-1 Return thấp             | Onboarding không đủ hấp dẫn |

# 21. KPI Anti-patterns

Không làm:

```txt
Chọn quá nhiều North Star Metrics.
Dùng page views làm success chính.
Dùng registered users thay cho verified users.
Tối ưu greetings mà bỏ qua block/report rate.
Tối ưu mentor request mà làm mentor quá tải.
Tối ưu posts mà để feed thành spam.
Tối ưu community count mà community không active.
Đo message volume mà không đo harassment/report.
Tạo dashboard trước khi chốt KPI.
Tracking event chứa dữ liệu nhạy cảm.
```

Một dashboard có 70 biểu đồ mà không ai biết cần hành động gì không phải data-driven. Nó là screensaver doanh nghiệp.
