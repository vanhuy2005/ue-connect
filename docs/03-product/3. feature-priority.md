# Feature Priority

## 1. Purpose

File này định nghĩa mức ưu tiên feature cho UEConnect.

Mục tiêu:

- Khóa MVP scope.
- Xác định feature nào bắt buộc làm trước.
- Tách rõ P0, P1, P2, P3 và Out of Scope.
- Giúp dev, designer, QA và product không kéo thêm feature lung tung vào MVP.
- Làm nền cho backlog, sprint planning, sitemap, feature specs và test planning.

Feature priority trả lời:

```txt
Feature nào bắt buộc có?
Feature nào nên có nếu còn thời gian?
Feature nào chỉ cần chuẩn bị kiến trúc?
Feature nào không làm trong MVP?
Feature nào bị cấm vì lệch product direction?
````

---

## 2. Priority Definition

| Priority | Meaning          | Product Decision                             |
| -------- | ---------------- | -------------------------------------------- |
| P0       | Bắt buộc cho MVP | Không có thì MVP chưa đạt                    |
| P1       | Rất quan trọng   | Nên có trong MVP pilot hoặc bản mở rộng gần  |
| P2       | Future-ready     | Chưa cần build sâu, nhưng nên chừa kiến trúc |
| P3       | Later roadmap    | Để sau khi product ổn định                   |
| OOS      | Out of Scope     | Không làm trong MVP                          |

---

## 3. Priority Principles

UEConnect ưu tiên feature theo thứ tự:

```txt
01. Trust trước growth
02. Verification trước social
03. Profile trước discovery
04. Greeting trước messaging
05. Safety trước scale content
06. Moderation trước community mở rộng
07. Permission trước admin power
08. Audit trước critical admin action
09. PWA-first trước native app
10. Student value trước monetization
```

Rule ngắn:

```txt
Nếu feature làm tăng social interaction nhưng chưa có safety/moderation tương ứng, feature đó không được lên P0.
```

# 11. Priority Decision Matrix

Khi có feature mới, dùng bảng này để quyết định.

| Question                                                     | If Yes | Priority Direction |
| ------------------------------------------------------------ | ------ | ------------------ |
| Feature có bắt buộc cho verification/trust không?            | Yes    | P0                 |
| Feature chặn rủi ro safety/privacy/security không?           | Yes    | P0                 |
| Feature cần để user tạo social value đầu tiên không?         | Yes    | P0                 |
| Feature cần cho meaningful connection không?                 | Yes    | P0/P1              |
| Feature giúp mentor/community nhưng không chặn MVP không?    | Yes    | P1                 |
| Feature cần kiến trúc nhưng chưa cần UI đầy đủ không?        | Yes    | P2                 |
| Feature chỉ là polish/social delight không?                  | Yes    | P2/P3              |
| Feature gây dating vibe không?                               | Yes    | OOS                |
| Feature biến app thành marketplace/job board/portal không?   | Yes    | OOS                |
| Feature cần legal/integration lớn chưa có stakeholder không? | Yes    | P3/OOS             |
