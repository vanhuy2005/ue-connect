# Onboarding

## 1. Feature Summary

### 1.1. One-line Description

Onboarding hướng dẫn user vừa được xác thực đi qua Welcome screen, hoàn thiện profile bắt buộc, hiểu quy định an toàn và khám phá các hành động đầu tiên trong UEConnect.

### 1.2. Feature Goal

Feature này tồn tại để:

- Chào mừng user sau khi verification được approved.
- Giải thích ngắn gọn UEConnect dùng để làm gì.
- Hướng user hoàn thiện profile trước khi vào full app.
- Giới thiệu các pillar chính: Feed, Discovery, Greeting, Messaging, Mentor, Community, Safety.
- Thiết lập privacy/default preferences ban đầu.
- Tạo activation checklist để user biết nên làm gì tiếp theo.
- Cá nhân hóa onboarding theo role: Student, Alumni, Advisor.
- Giảm cảm giác app trống, khó hiểu hoặc giống “portal trường”.
- Tăng tỷ lệ profile completion, first post, first greeting, first mentor request và community join.

### 1.3. Product Value

| Value | Description |
|---|---|
| User value | User hiểu nhanh UEConnect dùng để kết nối, học cùng, tìm mentor và tham gia cộng đồng |
| Product value | Tăng activation và giảm user bỏ app sau verification |
| Business value | Giúp MVP trông giống sản phẩm thật có funnel rõ ràng |
| Trust value | User hiểu vì sao cần verified identity, profile và privacy |
| Safety value | User được nhắc quy định cộng đồng, report/block và nội dung bị cấm |
| UX value | User có đường đi rõ ràng thay vì bị ném vào app như một linh hồn lạc lõng |

### 1.4. Non-goals

Feature này không làm:

- Không thay thế Profile Management.
- Không cho skip required profile fields.
- Không tạo onboarding quá dài như khóa học nhập môn.
- Không dùng dating language.
- Không dùng swipe/match/crush/hot để giới thiệu Discovery.
- Không ép user đăng bài/gửi lời chào ngay.
- Không dùng gamification quá trẻ con.
- Không làm tutorial toàn màn hình dài lê thê.
- Không yêu cầu user hoàn thành Mentor/Community trước khi vào app.
- Không làm dark mode onboarding trong MVP, nhưng layout phải future-ready.

## 2. Scope

### 2.1. In Scope

```txt
- Welcome screen after verification approval
- Role-based onboarding intro
- Required profile setup entry
- Activation checklist
- Safety/community guideline acknowledgement
- Privacy default confirmation
- First actions guidance
- Student onboarding
- Alumni onboarding
- Advisor onboarding
- Mentor onboarding entry if mentor_access exists or user requests mentor mode
- Skip optional onboarding steps
- Redirect logic based on profile completion
- Onboarding analytics events
- Mobile-first onboarding flow
````

### 2.2. Out of Scope

```txt
- Full product tour with many screens
- Video onboarding
- AI-generated onboarding
- Native mobile onboarding
- Dark mode onboarding
- Complex gamification
- Reward points
- Dating-like profile setup
- Full career pathway onboarding
- Full community manager training
- Admin onboarding
```

### 2.3. Future-ready

```txt
- Bilingual onboarding Vietnamese/English
- Personalized checklist based on behavior
- Mentor-specific onboarding path
- Club Manager onboarding path
- Progressive onboarding tips
- Product education cards
- Admin onboarding/training
- In-app coach marks
- A/B testing onboarding steps
```

### 4.2. Required Permissions

| Action                          | Required Role        | Required Permission                    | Scope |
| ------------------------------- | -------------------- | -------------------------------------- | ----- |
| View onboarding                 | verified active user | none                                   | own   |
| Complete onboarding             | verified active user | none                                   | own   |

| Enable mentor setup             | verified user        | `mentor_access` or mentor request flow | own   |
| View community prompt           | verified user        | none                                   | own   |

### 4.3. Access Rules

```txt
- Only active verified users can access onboarding.
- User with incomplete required profile must be routed to onboarding/profile setup.
- User can skip optional education cards.
- User cannot skip required profile fields.
- Suspended/banned users cannot access onboarding.
- Onboarding path should be role-aware.
- Account status overrides onboarding state.
- Profile completion state determines whether user can enter full app.
```

## 7. Onboarding Strategy

### 7.1. Design Direction

Onboarding should feel:

```txt
friendly
minimal
student-first
trusted
clear
mobile-first
not childish
not dating-like
not portal-like
```

Onboarding should not feel:

```txt
corporate training
long tutorial
dating app intro
school portal instruction page
AI-generated gradient deck
```

### 7.2. Onboarding Structure

UEConnect onboarding has 4 layers:

```txt
01. Welcome
02. Required Profile Setup
03. Safety & Privacy Confirmation
04. Activation Checklist
```

### 7.3. Skip Rules

| Step                    |             Can Skip? | Reason                          |
| ----------------------- | --------------------: | ------------------------------- |
| Welcome screen          |                   Yes | User can move quickly           |
| Required avatar         |                    No | Required for profile completion |
| Required profile fields |                    No | Needed for identity surface     |
| Safety guideline intro  | No/P0 acknowledgement | Needed for social safety        |
| Privacy defaults        |    No/P0 confirmation | Needed for trust                |
| Activation checklist    |                   Yes | Optional activation help        |
| First post prompt       |                   Yes | Optional                        |
| First greeting prompt   |                   Yes | Optional                        |
| Mentor prompt           |                   Yes | Optional                        |
| Community prompt        |                   Yes | Optional                        |

Rule:

```txt
User can skip education, not identity.
```

Đọc lên nghe hiển nhiên, nhưng app nào đó ngoài kia vẫn bắt user xem 8 slide rồi cho profile trống trơn. Loài người thật bền bỉ trong việc ưu tiên sai.

## 8. Role-based Onboarding

### 8.1. Student Onboarding

Student onboarding focuses on:

```txt
- hoàn thiện hồ sơ UEer
- hiểu Feed
- khám phá bạn cùng khoa/ngành/khóa
- gửi lời chào
- tìm mentor
- tham gia community/CLB
- hiểu report/block
```

Recommended copy:

```txt
Chào mừng bạn đến với UEConnect.
Đây là không gian để UEers kết nối, học cùng, tìm mentor và tham gia cộng đồng HCMUE.
```

### 8.2. Alumni Onboarding

Alumni onboarding focuses on:

```txt
- cập nhật HCMUE background
- thêm current field nếu muốn
- chia sẻ kinh nghiệm
- bật mentor mode nếu muốn hỗ trợ sinh viên
- tham gia alumni/community spaces
```

Recommended copy:

```txt
Chào mừng bạn trở lại cộng đồng HCMUE.
Bạn có thể chia sẻ kinh nghiệm, kết nối với UEers và hỗ trợ sinh viên qua mentor request.
```

### 8.3. Advisor Onboarding

Advisor onboarding focuses on:

```txt
- xác nhận department/position/support area
- thiết lập profile tư vấn
- cấu hình mentor/advising availability
- nhận mentor request từ sinh viên
- dùng messaging sau khi request được accepted
```

Recommended copy:

```txt
UEConnect giúp cố vấn và mentor hỗ trợ sinh viên theo một luồng rõ ràng, an toàn và có ngữ cảnh.
```

### 8.4. Mentor-enhanced Onboarding

If user has `mentor_access` or requests mentor mode:

```txt
- introduce mentor profile
- support topics
- availability
- request handling
- safety/reporting
```

Mentor onboarding should not make the product feel like LinkedIn.

## 9. Main User Flow

### 9.1. Happy Path: Newly Approved Student

```txt
Verification approved
→ User receives notification
→ User logs in
→ System detects approved + profile incomplete
→ Redirect to /app/onboarding
→ Show Student Welcome
→ Continue to Profile Setup
→ Upload avatar
→ Confirm display name, faculty, major, cohort
→ Add short bio
→ Select interests/goals
→ Confirm privacy defaults
→ Confirm safety basics
→ Profile becomes basic_complete
→ Show activation checklist
→ User enters /app/home
```

### 9.2. Happy Path: Alumni

```txt
Alumni verification approved
→ User opens onboarding
→ Show Alumni Welcome
→ Continue to Alumni Profile Setup
→ Upload avatar
→ Confirm HCMUE background
→ Add current field / can-help-with topics
→ Confirm privacy defaults
→ Optional mentor mode prompt
→ Show activation checklist
→ Enter app
```

### 9.3. Happy Path: Advisor

```txt
Advisor verification approved
→ User opens onboarding
→ Show Advisor Welcome
→ Confirm department and position
→ Add support area/advising topics
→ Confirm privacy defaults
→ Optional mentor availability setup
→ Show activation checklist
→ Enter app
```

### 9.4. Returning Incomplete User

```txt
User logs in
→ System checks profile completion
→ If incomplete: redirect onboarding/profile setup
→ User continues from last required step
→ Completes missing fields
→ Enters app
```

### 9.5. Optional Skip Flow

```txt
User opens optional activation checklist
→ User clicks Skip for now
→ System marks optional onboarding skipped
→ User enters /app/home
→ Checklist remains available from profile/home prompt
```

## 11. Alternative & Failure Flows

| Flow                                         | Condition                       | Expected Behavior                       |
| -------------------------------------------- | ------------------------------- | --------------------------------------- |
| User closes app during onboarding            | Partial progress                | Resume from required incomplete step    |
| User skips optional tutorial                 | Optional step skipped           | Allow app access if profile complete    |
| User skips avatar                            | Required missing                | Block completion with clear error       |
| User skips privacy confirmation              | Required missing                | Block completion                        |
| User skips safety confirmation               | Required missing                | Block completion                        |
| Profile save fails                           | Server/network issue            | Preserve form input and allow retry     |
| Avatar upload fails                          | Invalid/failed upload           | Show retry and file rules               |
| Account suspended during onboarding          | Account status changed          | Redirect restricted account             |
| Role changes during onboarding               | Admin correction/reverification | Reload role-based onboarding            |
| User already completed onboarding            | Opens onboarding route          | Redirect home or show checklist summary |
| User tries app route with incomplete profile | Guard catches                   | Redirect profile setup/onboarding       |

## 12. Onboarding Steps

### 12.1. Step 1 — Welcome

| Field      | Description                                      |
| ---------- | ------------------------------------------------ |
| Purpose    | Confirm user is verified and introduce UEConnect |
| Required   | Yes but can move forward immediately             |
| Role-aware | Yes                                              |
| UI         | Simple welcome card                              |
| CTA        | Bắt đầu hoàn thiện hồ sơ                         |

Content should include:

```txt
- You are verified
- UEConnect is HCMUE-only
- Main value: kết nối, học cùng, mentor, community
- Next step: complete profile
```

### 12.2. Step 2 — Required Profile Setup

Handled by `profile-management.md`.

Required:

```txt
avatar
display_name
role-specific fields
short_bio
privacy defaults
```

### 12.3. Step 3 — Safety Basics

User must acknowledge:

```txt
- Không spam
- Không quấy rối
- Không dating/sexual content
- Không giả mạo danh tính
- Không lộ thông tin cá nhân người khác
- Không chia sẻ tài liệu vi phạm bản quyền
- Có thể report/block khi cần
```

### 12.4. Step 4 — Privacy Defaults

User confirms:

```txt
- Profile visible to verified UEers by default
- Full MSSV is never public
- Email/phone hidden by default
- Discovery visibility can be changed
- User can block/report
```

### 12.5. Step 5 — Activation Checklist

Checklist can include:

```txt
- Hoàn thiện hồ sơ
- Đọc quy định cộng đồng
- Viết bài đầu tiên
- Gửi lời chào đầu tiên
- Khám phá mentor
- Tham gia cộng đồng
```

Only first two or three should be strongly encouraged. Do not make the app feel like homework. Sinh viên đã có đủ bài tập rồi, cảm ơn.

## 20. Security & Privacy

### 20.1. Security Rules

```txt
- Only authenticated active verified users can access onboarding.
- Required profile fields must be validated server-side.
- Safety/privacy acknowledgement must be stored server-side.
- Do not allow onboarding skip to bypass profile completion.
- Account status must be checked on every protected route.
- Role-based onboarding content must come from server-side role.
```

### 20.2. Privacy Rules

```txt
- Privacy defaults must be explained before public profile exposure.
- Full MSSV is never shown in onboarding public preview.
- Email/phone hidden by default.
- Analytics must not store raw bio or sensitive profile fields.
- Onboarding progress is user-private.
```

### 20.3. Sensitive Data

| Data                   |   Sensitive? | Handling              |
| ---------------------- | -----------: | --------------------- |
| Onboarding progress    | Low/moderate | Owner/admin only      |
| Privacy choices        |     Moderate | Owner/admin if needed |
| Safety acknowledgement | Low/moderate | Owner/admin if needed |
| Profile draft          |     Moderate | Owner only            |
| MSSV                   |          Yes | Never public          |
| Email/phone            |          Yes | Hidden by default     |

## 24. Edge Cases

| Edge Case                                             | Expected Handling                            |
| ----------------------------------------------------- | -------------------------------------------- |
| User closes app during onboarding                     | Resume from required incomplete step         |
| User skips optional cards                             | Allow app access if required steps complete  |
| User tries to skip profile setup                      | Block and redirect back                      |
| User completes onboarding but profile incomplete      | Keep restricted to setup                     |
| User completes profile but not safety acknowledgement | Show safety step                             |
| User completes safety but not privacy                 | Show privacy step                            |
| User account suspended during onboarding              | Redirect account restricted                  |
| User role changes after admin correction              | Re-resolve onboarding path                   |
| User opens old onboarding URL after completion        | Redirect home or checklist summary           |
| Network fails during acknowledgement                  | Show retry, do not mark complete             |
| Checklist event fails                                 | Do not block user; retry silently if safe    |
| Privacy settings fail to save                         | Do not complete onboarding                   |
| User opens mentor prompt without mentor access        | Hide prompt or show request mentor mode flow |
| User opens alumni onboarding as student               | Redirect to correct role path                |
