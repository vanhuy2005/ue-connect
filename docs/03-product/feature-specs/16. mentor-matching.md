# Mentor Matching

## 1. Feature Summary

### 1.1. One-line Description

Mentor Matching cho phép verified students tìm kiếm, khám phá và gửi mentor request đến alumni/advisors/approved mentors dựa trên topic, question, goal, urgency, mentor expertise và career pathway context.

### 1.2. Feature Goal

Feature này tồn tại để:

- Giúp sinh viên HCMUE tìm được người phù hợp để hỏi định hướng học tập, nghề nghiệp, kỹ năng, thực tập, portfolio hoặc kinh nghiệm cá nhân.
- Cho alumni, advisor hoặc approved mentors chia sẻ kinh nghiệm theo cách có kiểm soát.
- Tạo một mentor flow riêng, không bị lẫn với Discovery thông thường.
- Hỗ trợ mentor request có cấu trúc rõ: topic, question, goal, urgency.
- Cho mentor accept, decline, ask more info hoặc pause availability.
- Khi request được accepted, hệ thống tạo conversation.
- Hỗ trợ scheduling ở mức future-ready/P1.
- Hỗ trợ rating/feedback thông minh, không biến mentor thành sản phẩm thương mại.
- Tích hợp với Career Pathway để user khám phá định hướng nghề nghiệp.
- Tích hợp với notification, messaging, safety reporting và moderation.
- Không tạo recruiter portal.
- Không biến mentor thành LinkedIn clone quá cứng nhắc.

### 1.3. Product Value

| Value | Description |
|---|---|
| User value | Sinh viên có thể tìm người đã có kinh nghiệm để hỏi đúng vấn đề |
| Mentor value | Alumni/advisor/mentor có cách đóng góp cho cộng đồng HCMUE mà không bị spam |
| Product value | UEConnect có growth-oriented pillar, không chỉ feed/chat làm màu |
| Business value | Đây là feature portfolio rất mạnh vì giải quyết pain point thật của sinh viên |
| Trust value | Mentor access phải được approved hoặc admin granted |
| Safety value | Mentor request có report/block/moderation |
| UX value | Structured request giúp mentor hiểu vấn đề nhanh, không phải đoán ý như bói toán giao diện |

### 1.4. Non-goals

Feature này không làm:

- Không tạo recruiter portal.
- Không tạo employer role.
- Không tạo job board chính thức trong MVP.
- Không cho user tự nhận là mentor nếu chưa được approve/grant.
- Không biến mentor rating thành sao 1-5 kiểu mua hàng.
- Không public private mentor conversation.
- Không cho student spam mentor request.
- Không đảm bảo mentor bắt buộc phải phản hồi.
- Không tự động matching bằng AI như quyết định cuối cùng trong MVP.
- Không thay thế cố vấn học tập chính thức của nhà trường.
- Không biến mentor thành tư vấn pháp lý, tài chính, sức khỏe chuyên môn.
- Không dùng ngôn ngữ quá corporate như LinkedIn Recruiter Lite, vì xin đừng.

## 4. Actors & Permissions

### 4.1. Actors

| Actor                 | Description                                               |
| --------------------- | --------------------------------------------------------- |
| Student               | Main mentee. Sends mentor requests                        |
| Alumni                | Can become mentor after approval/grant                    |
| Advisor               | Can become mentor/advisor mentor after verification/grant |
| Approved Mentor       | User with active mentor access                            |
| Admin                 | Reviews/grants/revokes mentor access                      |
| Mentor Reviewer       | Admin-like scoped reviewer for mentor applications        |
| Moderator             | Reviews reported mentor profiles/requests/messages        |
| Suspended/Banned User | Cannot use mentor features                                |

### 4.2. Who Can Become Mentor

Recommended MVP rule:

```txt
- Alumni can apply/be granted mentor access.
- Advisor can apply/be granted mentor access.
- Student mentor is not default in MVP.
- Student mentor can be allowed later only by admin-granted exceptional permission.
```

Reason:

```txt
Mentor should signal trust and experience. Nếu ai cũng bật mentor được thì chữ mentor lập tức mất giá như coupon giảm 5%.
```

### 4.3. Mentor Access Rules

| User Role        | Can Request Mentor Access? | Can Be Admin Granted? | Notes                    |
| ---------------- | -------------------------: | --------------------: | ------------------------ |
| Student          | No by default/P2 exception |      Yes, exceptional | Future peer mentor       |
| Alumni           |                        Yes |                   Yes | Primary mentor pool      |
| Advisor          |                        Yes |                   Yes | Academic/career guidance |
| Admin            |          Not as admin role |   Yes as user account | Separate concern         |
| Suspended/Banned |                         No |                    No | Restricted               |

### 4.4. Required Permissions

| Action                 | Required Actor          | Required Permission    | Scope            |
| ---------------------- | ----------------------- | ---------------------- | ---------------- |
| View mentor discovery  | verified active user    | none                   | app              |
| View mentor profile    | verified active user    | none                   | privacy-filtered |
| Request mentor access  | eligible alumni/advisor | none                   | own              |
| Approve mentor access  | admin/reviewer          | `manage_mentor_access` | global           |
| Reject mentor access   | admin/reviewer          | `manage_mentor_access` | global           |
| Grant mentor directly  | admin                   | `manage_mentor_access` | global           |
| Revoke mentor access   | admin                   | `manage_mentor_access` | global           |
| Create mentor profile  | approved mentor         | none                   | own              |
| Update mentor profile  | approved mentor         | none                   | own              |
| Pause availability     | approved mentor         | none                   | own              |
| Send mentor request    | verified student/user   | none                   | target mentor    |
| Accept mentor request  | target mentor           | none                   | own request      |
| Decline mentor request | target mentor           | none                   | own request      |
| Ask more information   | target mentor           | none                   | own request      |
| Report mentor request  | participant             | none                   | own request      |
| Moderate mentor abuse  | moderator/admin         | `moderate_content`     | global/scoped    |

### 4.5. Access Rules

```txt
- Only verified active users can view mentor discovery.
- Only active approved mentors appear in mentor discovery.
- Mentor must have mentor_visibility enabled.
- Mentor must not be suspended/banned/hidden by moderation.
- Blocked users cannot see/request each other.
- Student cannot send duplicate pending mentor request to same mentor.
- Mentor can accept/decline/ask more info only their own received requests.
- Accepted mentor request creates conversation.
- Mentor can pause availability, then new requests are disabled.
- Admin can revoke mentor access with reason and audit.
```

## 7. Mentor Access Model

### 7.1. Mentor Access Meaning

Mentor access là capability cho phép user:

```txt
- xuất hiện trong mentor discovery
- tạo mentor profile
- nhận mentor request
- accept/decline/ask more info
- quản lý availability
```

Mentor access không phải:

```txt
- system primary role
- recruiter role
- admin role
- proof of employment
- official school advisor unless verified as advisor
```

### 7.2. Mentor Access Statuses

| Status                  | Meaning                                        |
| ----------------------- | ---------------------------------------------- |
| `not_requested`         | User chưa yêu cầu mentor access                |
| `requested`             | User đã gửi yêu cầu                            |
| `under_review`          | Admin/reviewer đang xem                        |
| `need_more_information` | Admin yêu cầu bổ sung                          |
| `approved`              | Mentor access active                           |
| `rejected`              | Yêu cầu bị từ chối                             |
| `paused`                | Mentor tự tạm dừng nhận request                |
| `revoked`               | Admin thu hồi mentor access                    |
| `suspended`             | Mentor access tạm hạn chế do safety/moderation |

### 7.3. Mentor Access Request Fields

| Field                         |    Required | Description                            |

| `mentor_intro`                |         Yes | Giới thiệu ngắn vì sao muốn mentor     |

| `experience_summary`          |         Yes | Kinh nghiệm học tập/nghề nghiệp        |
| `current_position_or_context` | Conditional | Công việc/lĩnh vực hiện tại nếu alumni |
| `advisor_area`                | Conditional | Mảng tư vấn nếu advisor                |
| `portfolio_or_profile_link`   |          No | Link minh chứng nếu có                 |
| `availability_note`           |         Yes | Mức độ sẵn sàng                        |

| `admin_note`                  |          No | Internal admin note                    |

### 7.4. Admin Mentor Access Actions

| Action                  | Effect                                     |      Reason Required | Audit |
| ----------------------- | ------------------------------------------ | -------------------: | ----: |
| `approve`               | Mentor access approved                     |                  Yes |   Yes |

## 10. Mentor Request Rules

### 10.1. Send Request Rules

```txt
- Sender must be verified active user.
- Target must be approved active mentor.
- Target mentor availability must allow new requests.
- Sender cannot send request to self.
- Sender cannot send request to blocked mentor.
- Mentor cannot have blocked sender.
- Sender cannot have duplicate pending request to same mentor.
- Sender must fill topic, question, goal, urgency.
- Request content must follow safety/content policy.
```

### 10.2. Pending Limit Rules

Recommended MVP/P1:

```txt
Per student:
- max pending mentor requests total = 5
- max pending requests to same mentor = 1

Per mentor:
- max pending requests = configurable, default 10
```

When mentor limit reached:

```txt
availability_status = full
new request disabled
```

### 10.3. Accept Rules

```txt
- Only target mentor can accept.
- Request must be pending or updated_by_student.
- Mentor must still have active mentor access.
- No block can exist between participants.
- Accepted request creates conversation.
- Notification sent to student.
```

### 10.4. Decline Rules

```txt
- Only target mentor can decline.
- Decline should allow optional user-safe reason.
- Declined request cannot be resent immediately.
- Sender may try another mentor.
```

### 10.5. Ask More Information Rules

```txt
- Only target mentor can ask more information.
- Mentor must provide instruction.
- Request status becomes need_more_information.
- Student can update request.
- Mentor receives notification when updated.
```

### 10.6. Pause Availability Rules

```txt
- Mentor can pause own availability.
- Paused mentor does not receive new requests.
- Existing pending requests remain unless mentor chooses to decline/cancel by policy.
- Admin can suspend mentor access separately.
```

## 11. Matching / Ranking MVP

### 11.1. Matching Goal

Mentor discovery should recommend mentors based on:

```txt
- topic relevance
- career pathway relevance
- expertise overlap
- same faculty/major context
- mentor availability
- mentor request load
- profile completeness
- trust status
- past positive feedback signals
```

Not based on:

```txt
- popularity only
- number of requests only
- attractiveness
- social clout
- vanity metrics
```

### 11.2. MVP Ranking Inputs

| Signal                 | Meaning                                  | Direction             |
| ---------------------- | ---------------------------------------- | --------------------- |
| Topic match            | Mentor expertise matches student topic   | Strong positive       |
| Career pathway match   | Mentor linked to career pathway          | Strong positive       |
| Same faculty           | Shared faculty                           | Medium positive       |
| Same major             | Shared major                             | Medium positive       |
| Alumni context         | Alumni in relevant field                 | Positive              |
| Advisor context        | Advisor expertise relevant               | Positive              |
| Availability           | Available mentors ranked higher          | Strong positive       |
| Pending load           | Too many pending requests lowers ranking | Negative              |
| Profile completeness   | Better profile ranked higher             | Positive              |
| Report/moderation risk | Risk lowers/excludes                     | Negative/hard exclude |
| Block relation         | Exclude                                  | Hard exclude          |

### 11.3. Scoring Direction

```txt
score =
  topic_match_score
+ career_pathway_score
+ expertise_overlap_score
+ hcmue_context_score
+ availability_score
+ profile_quality_score
- workload_penalty
- report_risk_penalty
```

### 11.4. Safe Matching Explanation

Allowed:

```txt
- Phù hợp với chủ đề Frontend
- Có kinh nghiệm về Portfolio
- Cùng khoa
- Liên quan đến Career Pathway: Software Engineering
- Đang nhận mentor request
```

Avoid:

```txt
- Mentor này được nhiều người thích nhất
- Mentor này hot nhất
- Mentor này phản hồi bạn vì bạn phù hợp 98%
- Mentor này vừa online 2 phút trước
```

## 15. Main User Flows

### 15.1. Student Finds Mentor

```txt
Student opens Mentor
→ Selects topic/career pathway/filter
→ System shows mentor cards
→ Student opens mentor profile
→ Student clicks Gửi yêu cầu mentor
→ Fills topic/question/goal/urgency
→ Submit request
→ Mentor receives notification
```

### 15.2. Mentor Accepts Request

```txt
Mentor opens notification/request dashboard
→ Reviews request
→ Clicks Chấp nhận
→ System validates mentor status
→ Request status = accepted
→ Conversation created
→ Student notified
→ Mentor/student can chat
```

### 15.3. Mentor Declines Request

```txt
Mentor opens request
→ Clicks Từ chối
→ Optional user-safe reason
→ Request status = declined
→ Student sees status/notification
```

### 15.4. Mentor Asks More Information

```txt
Mentor opens request
→ Clicks Cần thêm thông tin
→ Writes instruction
→ Request status = need_more_information
→ Student notified
→ Student updates request
→ Request status = updated_by_student
→ Mentor notified
```

### 15.5. Mentor Pauses Availability

```txt
Mentor opens Mentor Dashboard
→ Toggle Tạm dừng nhận yêu cầu
→ availability_status = paused
→ Mentor hidden/disabled from new requests
```

### 15.6. Admin Approves Mentor Access

```txt
User requests mentor access
→ Admin opens mentor access queue
→ Reviews profile/context
→ Approve/reject/need more info
→ Reason required
→ User notified
→ Audit created
```

### 15.7. Report Mentor Request

```txt
Student/Mentor opens mentor request
→ Selects Báo cáo
→ Chooses reason
→ System creates report
→ Auto-block if safety policy applies
→ Request becomes restricted/reported
→ Moderation queue receives report
```

## 22. Security & Privacy

### 22.1. Security Rules

```txt
- Mentor access cannot be self-granted.
- Mentor discovery only shows approved active mentors.
- Mentor request creation is server-authorized.
- Request accept/decline/more info only by target mentor.
- Student can edit/update only own request when status allows.
- Mentor conversation creation must be transactional.
- Block prevents mentor request and conversation.
- Suspended/banned users cannot use mentor features.
- Mentor profile public fields are sanitized.
- Admin mentor access actions require permission, reason and audit.
```

### 22.2. Privacy Rules

```txt
- Mentor profile shows only public-safe fields.
- Student request visible only to student, target mentor and authorized admin/moderator if reported.
- Feedback is private/aggregate, not public marketplace review.
- Mentor request attachments follow restricted access.
- Do not expose email/phone/MSSV/evidence.
- Notification preview should not include full private question if sensitive.
```

| Data                         |         Sensitive? | Handling             |
| ---------------------------- | -----------------: | -------------------- |
| Mentor request question      | Moderate/Sensitive | Participants only    |
| Student goal/context         |           Moderate | Participants only    |
| Attachments                  |          Sensitive | Protected access     |
| Mentor feedback              |          Sensitive | Private/aggregate    |
| Mentor access admin notes    |          Sensitive | Admin only           |

## 26. Edge Cases

| Edge Case                                   | Expected Handling                       |
| ------------------------------------------- | --------------------------------------- |
| Student sends request to self               | Reject                                  |
| Mentor access revoked while request pending | Disable accept, show unavailable        |
| Mentor pauses after student opens form      | Submit rejected with unavailable state  |
| Mentor pending limit reached                | Disable new request                     |
| Student pending limit reached               | Prevent new request                     |
| Duplicate pending request                   | Show existing pending state             |
| Mentor accepts same request twice           | Idempotent, no duplicate conversation   |
| Conversation creation fails                 | Transaction rollback or safe retry      |
| Student updates request after accepted      | Reject                                  |
| Mentor asks more info after declined        | Reject stale action                     |
| Student cancels while mentor reviewing      | Mentor action rejected with stale state |
| Mentor gets suspended                       | Hide mentor profile, restrict requests  |
| Student gets suspended                      | Prevent request/update                  |
| Block created between users                 | Request/conversation restricted         |
| Report mentor request                       | Report + auto-block if policy applies   |
| Mentor profile hidden by moderation         | Exclude from discovery                  |
| Feedback submitted twice                    | Prevent duplicate feedback              |
| Browser push denied                         | In-app notification still exists        |
| Career pathway deleted                      | Mentor profile/request shows fallback   |
| Attachment removed                          | Show unavailable attachment             |
