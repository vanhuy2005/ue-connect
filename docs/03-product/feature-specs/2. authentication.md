# Authentication

## 1. Feature Summary

### 1.1. One-line Description

Authentication cho phép user đăng ký, đăng nhập, đăng xuất và được điều hướng đúng trạng thái tài khoản trước khi truy cập UEConnect, với ưu tiên đăng nhập bằng email HCMUE thuộc hệ sinh thái Microsoft 365 / Outlook thông qua Microsoft Entra ID.

### 1.2. Feature Goal

Feature này tồn tại để:

- Cho phép user tạo tài khoản bằng email HCMUE hợp lệ.
- Cho phép user đăng nhập bằng email và mật khẩu nếu chưa dùng SSO.
- Hỗ trợ đăng nhập nhanh bằng Microsoft Entra ID / Outlook / Microsoft 365 nếu trường cấp mail theo hệ Microsoft.
- Chặn user chưa xác thực truy cập app chính.
- Điều hướng user theo trạng thái account: pending, rejected, need more information, suspended, banned, profile incomplete, active.
- Bảo vệ admin routes bằng authentication, account status guard và permission guard.
- Chuẩn bị khả năng localization UI tiếng Việt/English trong tương lai.
- Tạo nền cho Verification Identity, Profile Setup, Admin Operations và toàn bộ app shell.

### 1.3. Product Value

| Value | Description |
|---|---|
| User value | User có thể đăng nhập bằng email HCMUE quen thuộc, đặc biệt nếu email trường dùng Outlook/Microsoft 365 |
| Product value | Tạo cổng vào duy nhất, rõ ràng và đáng tin cho verified social platform |
| Business value | Đảm bảo UEConnect chỉ phục vụ cộng đồng HCMUE |
| Trust value | Chặn email ngoài phạm vi trường ngay từ bước đăng ký |
| Security value | Tạo nền cho route guard, account restriction, admin access, session control và SSO |
| UX value | User luôn được redirect đến đúng bước tiếp theo, không bị lạc trong app |

### 1.4. Non-goals

Feature này không làm:

- Không dùng Google OAuth làm phương án chính nếu HCMUE sử dụng Outlook/Microsoft 365.
- Không cho đăng ký bằng username.
- Không cho đăng ký bằng email ngoài domain HCMUE.
- Không dùng phone number login trong MVP.
- Không dùng anonymous account.
- Không dùng Facebook/TikTok/GitHub social login.
- Không triển khai 2FA trong MVP.
- Không thay thế Verification Identity. Có email HCMUE không đồng nghĩa đã verified đầy đủ.
- Không tự động cấp full app access chỉ vì Microsoft SSO thành công.
- Không đồng bộ dữ liệu học vụ chính thức nếu chưa có quyền tích hợp từ trường.

## 2. Scope

### 2.1. In Scope

```txt
- Register with HCMUE email
- Login with email and password
- Logout
- Remember me
- Password validation
- Password reset / forgot password
- Account status routing
- Verified-only app route guard
- Admin route auth guard
- Microsoft Entra ID / Outlook SSO login direction
- Allowed email domain validation
- Session expiration handling
- Suspended/banned account restriction
- Localization-ready auth copy
- Basic authentication analytics events
````

### 2.2. Out of Scope

```txt
- Username login
- Phone OTP login
- Anonymous login
- Google OAuth as primary login for HCMUE Outlook account
- Facebook/TikTok/GitHub social login
- Two-factor authentication in MVP
- Device management page
- Advanced login rate limiting in MVP
- Full SIS / student database integration
- Multi-university login
```

### 2.3. Future-ready

```txt
- 2FA
- Login rate limiting hardening
- Device/session management
- Official Microsoft Entra tenant integration with HCMUE
- Microsoft Graph profile enrichment if approved
- Login anomaly detection
- Account recovery hardening
- Security notification for suspicious login
- Role hints from Entra groups if school grants access
```

### 4.2. Required Permissions

| Action                    | Required Role          | Required Permission    | Scope         |
| ------------------------- | ---------------------- | ---------------------- | ------------- |
| Register                  | guest                  | none                   | public        |
| Login                     | guest                  | none                   | public        |
| Microsoft SSO login       | guest                  | none                   | public        |

| Forgot password           | guest                  | none                   | public        |

| Access app shell          | verified active user   | none                   | own           |

| Access admin moderation   | admin/moderator        | `moderate_content`     | global/scoped |
| Access admin permissions  | admin                  | `manage_permissions`   | global        |

### 4.3. Access Rules

```txt
- User can only register with allowed HCMUE email domain.
- User can login using email and password.
- User can login using Microsoft Entra ID / Outlook SSO when configured.
- Username is not supported.
- Password minimum length is 8 characters.
- Special character is not required.
- Remember me is supported.
- 2FA is not required in MVP.
- Login rate limit is not required in MVP, but route should be easy to harden later.
- Account status always overrides role and permission.
- Admin routes require authentication + active account + admin role/permission.
- UI hiding is not authorization.
```

## 7. Authentication Method Decision

### 7.1. Recommended Laravel Auth Stack

UEConnect should use:

```txt
Laravel Fortify + custom Blade UI
```

Reason:

* Auth action backend rõ.
* Dễ customize redirect theo account status.
* Không bị Jetstream thêm nhiều structure không cần.
* Phù hợp Laravel + Blade + TailwindCSS + Livewire + PWA.
* Dễ tách UI tiếng Việt nhưng code English.

Alternative:

```txt
Laravel Breeze nếu muốn scaffold nhanh hơn.
```

Breeze phù hợp nếu ưu tiên tốc độ, nhưng khi cần Microsoft SSO, route guard, custom redirect và account state phức tạp, Fortify + custom UI sạch hơn.

### 7.2. Microsoft SSO Provider

Nếu HCMUE email dùng Outlook/Microsoft 365, hướng đúng là:

```txt
Microsoft Entra ID
OAuth 2.0 Authorization Code Flow
OpenID Connect
```

Recommended Laravel package direction:

```txt
Laravel Socialite
SocialiteProviders/Microsoft
or
SocialiteProviders/Microsoft-Azure
```

Tên “Azure AD” cũ đã được Microsoft đổi branding thành Microsoft Entra ID. Nhưng trong package/community docs vẫn có thể thấy `microsoft-azure` hoặc `azure`. Vui vẻ chưa, chỉ riêng tên dịch vụ cũng đủ làm tài liệu auth già đi trước tuổi.

### 7.3. Microsoft SSO Does and Does Not Do

Microsoft SSO does:

```txt
- Prove user can access a Microsoft account/email.
- Return identity claims such as email/name depending on tenant and scopes.
- Improve login UX.
- Reduce password friction.
```

Microsoft SSO does not automatically:

```txt
- Prove user is currently a valid student.
- Prove MSSV.
- Prove alumni status.
- Prove advisor role.
- Replace admin verification.
- Grant full UEConnect access.
```

Rule:

```txt
Microsoft SSO authenticates account ownership.
Verification Identity approves UEConnect role and product access.
```

## 8. Email & Domain Policy

### 8.1. Allowed User Email Domain

Only users with HCMUE email can register.

```txt
allowed_user_email_domains:
- hcmue.edu.vn
```

### 8.2. Advisor / Teacher Domain

Advisor/teacher email domain should be configurable.

```txt
allowed_advisor_email_domains:
- hcmue.edu.vn
- additional official teacher/advisor domains when confirmed
```

Do not hard-code unknown domains. Use config or database setting.

### 8.3. Domain Validation Rules

| Case                                     | Behavior                                       |
| ---------------------------------------- | ---------------------------------------------- |
| Email domain is allowed                  | Continue registration                          |
| Email domain is not allowed              | Reject with clear error                        |
| Email already exists                     | Show duplicate account error                   |
| Microsoft SSO email not verified/trusted | Reject                                         |
| Microsoft SSO email domain mismatch      | Reject                                         |
| User changes email after verification    | Requires future re-verification / admin review |

### 8.4. Email Normalization

```txt
- Trim spaces.
- Lowercase email.
- Validate domain after normalization.
- Store normalized email.
```

Example:

```txt
NguyenVanA@HCMUE.EDU.VN → nguyenvana@hcmue.edu.vn
```

### 8.5. UI Copy

Good:

```txt
UEConnect hiện chỉ hỗ trợ email thuộc HCMUE.
```

For Microsoft SSO button:

```txt
Tiếp tục với Outlook HCMUE
```

Alternative:

```txt
Đăng nhập bằng tài khoản Microsoft HCMUE
```

Avoid:

```txt
Đăng nhập bằng Google
```

vì sai platform, sai mental model, và nghe như product đang mặc áo trường khác.

## 9. Main User Flow

### 9.1. Register with Email + Password

```txt
Guest opens register page
→ Enters HCMUE email and password
→ System normalizes email
→ System validates email domain
→ System validates password
→ System creates account
→ Account status = registered
→ Redirect to verification start
```

### 9.2. Login with Email + Password

```txt
Guest opens login page
→ Enters email and password
→ Optionally chooses Remember me
→ System validates credentials
→ System checks account status
→ Redirect based on account status
```

### 9.3. Microsoft Entra ID / Outlook SSO Login

```txt
Guest clicks "Tiếp tục với Outlook HCMUE"
→ System redirects to Microsoft identity platform authorization endpoint
→ User signs in with HCMUE Microsoft account
→ Microsoft redirects back to UEConnect callback URL
→ System validates state and authorization code
→ System exchanges code for tokens
→ System validates ID token / claims
→ System extracts email / unique provider subject
→ System validates allowed domain
→ If account exists: link or login
→ If account does not exist: create account
→ System checks verification status
→ Redirect to verification or app
```

### 9.4. Logout Flow

```txt
Authenticated user clicks logout
→ System invalidates local session
→ System redirects to login or landing
```

Optional future:

```txt
Also redirect to Microsoft logout endpoint if full SSO sign-out is desired.
```

### 9.5. Forgot Password Flow

```txt
Guest opens forgot password
→ Enters HCMUE email
→ System sends reset link if account exists
→ User opens reset link
→ Enters new password
→ System updates password
→ User can login
```

Security copy rule:

```txt
Do not reveal whether an email exists during forgot password.
```

## 10. Redirect Logic

### 10.1. After Login Redirect

```txt
if account_status in [banned, suspended]:
    redirect /system/account-restricted

else if verification_status not approved:
    redirect /verification/status

else if profile is incomplete:
    redirect /app/onboarding or /app/profile/setup

else:
    redirect /app/home
```

### 10.2. Admin Redirect

```txt
If admin logs in from normal login:
    do not force redirect to /admin

If admin tries to access /admin:
    allow only if account active + required admin permission

If admin lacks permission:
    show /system/403
```

### 10.3. Route Guard Matrix

| User State                         | `/auth/login`       | `/verification/status`       | `/app/home`         | `/admin`         |
| ---------------------------------- | ------------------- | ---------------------------- | ------------------- | ---------------- |
| Guest                              | Allowed             | Redirect login               | Redirect login      | Redirect login   |
| Registered                         | Redirect status     | Allowed                      | Redirect status     | Denied           |
| Pending verification               | Redirect status     | Allowed                      | Redirect status     | Denied           |
| Need more information              | Redirect status     | Allowed                      | Redirect more-info  | Denied           |
| Rejected                           | Redirect status     | Allowed                      | Redirect rejected   | Denied           |
| Active verified profile incomplete | Redirect onboarding | Allowed if needed            | Redirect onboarding | Permission-based |
| Active verified ready              | Redirect home       | Redirect home/status summary | Allowed             | Permission-based |
| Suspended                          | Restricted          | Restricted                   | Restricted          | Restricted       |
| Banned                             | Restricted          | Restricted                   | Restricted          | Restricted       |

## 13. Session & Remember Me

### 13.1. Remember Me

Remember me is supported.

Rules:

```txt
- User can select Remember me on email/password login.
- Remembered session still respects current account status.
- If user becomes suspended/banned, next request must be restricted.
- Remember me does not bypass verification.
```

### 13.2. Microsoft SSO Session

UEConnect maintains its own local Laravel session after Microsoft SSO.

```txt
Microsoft session != UEConnect session.
```

Optional future:

```txt
Full SSO logout may sign out from Microsoft account too,
but MVP can only destroy UEConnect local session.
```

### 13.3. Session Expiry

If session expires:

```txt
- Redirect to login.
- Preserve intended URL if safe.
- Do not preserve admin destructive action URLs.
```

### 13.4. Account Status Change While Session Active

If account status changes during active session:

```txt
- Next request must check current account_status.
- Suspended/banned user must be redirected to restricted account page.
- Permission revocation must take effect on next protected action.
```

```txt

- Do not keep long-lived provider tokens.
```

## 21. Security & Privacy

### 21.1. Security Rules

```txt
- Validate email domain server-side.
- Do not rely on client-side email validation.
- Hash passwords using Laravel default secure hashing.
- Invalidate session on logout.
- Regenerate session after login.
- Regenerate CSRF token as appropriate.
- Validate OAuth/OIDC state parameter.
- Validate Microsoft identity claims.
- Prefer authorization code flow.
- Do not store Microsoft tokens unless needed.
- Do not reveal whether email exists during password reset.
- Do not expose account restriction internals publicly.
- Admin routes must check permission server-side.
```

### 21.2. Privacy Rules

```txt
- Do not show email publicly by default.
- Do not log password or reset token.
- Do not send password/reset token to analytics.
- Do not leak account status details to unrelated users.
- Do not expose admin permission names to normal users.
- Do not store Microsoft refresh token unless there is a real Graph use case.
```

### 21.3. Sensitive Data

| Data                   | Sensitive? | Handling               |
| ---------------------- | ---------: | ---------------------- |
| Email                  |        Yes | Private by default     |
| Password               |        Yes | Hashed only            |
| Remember token         |        Yes | Framework-managed      |
| Reset token            |        Yes | Token table, never log |
| Session ID             |        Yes | Protected              |
| Login IP               |   Moderate | Admin/security only    |
| Microsoft provider ID  |        Yes | Internal only          |
| Microsoft tenant ID    |   Moderate | Internal/config        |

## 25. Edge Cases

| Edge Case                                                   | Expected Handling                                     |
| ----------------------------------------------------------- | ----------------------------------------------------- |

| User logs in while banned                                   | Redirect `/system/account-restricted`                 |
| Active verified user profile incomplete                     | Redirect onboarding/profile setup                     |
| Admin logs in from normal login                             | Do not auto-force `/admin`                            |
| Admin opens `/admin` without permission                     | 403                                                   |
| Account status changes while session active                 | Next request enforces new status                      |

| Microsoft SSO-only account has no password                  | Allow SSO login; password setup future                |
| Login route accessed by active user                         | Redirect based on status                              |
| `/app/*` accessed by guest                                  | Redirect login                                        |
| `/app/*` accessed by unverified                             | Redirect verification status                          |
