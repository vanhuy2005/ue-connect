# Post & Comment

## 1. Feature Summary

### 1.1. One-line Description

Post & Comment cho phép verified users tạo bài viết, xem chi tiết bài viết, bình luận, phản hồi một cấp, chỉnh sửa/xóa nội dung của mình, tương tác với nội dung và báo cáo nội dung vi phạm trong UEConnect.

### 1.2. Feature Goal

Feature này tồn tại để:

- Cho user chia sẻ nội dung trong cộng đồng HCMUE.
- Cho user thảo luận thông qua comment và reply một cấp.
- Hỗ trợ bài viết text và image.
- Hỗ trợ visibility để kiểm soát phạm vi hiển thị.
- Hỗ trợ reaction/save để tăng tương tác nhưng không biến app thành cuộc đua dopamine rẻ tiền.
- Hỗ trợ edit/delete own content với rules rõ ràng.
- Hỗ trợ report/block/moderation ngay tại post/comment.
- Đảm bảo nội dung vi phạm có thể bị hidden/deleted/restored bởi moderator/admin.
- Không hỗ trợ anonymous/confession/dating content.
- Tạo nền cho Feed, Community, Mentor insight, Career content và Notification.

### 1.3. Product Value

| Value | Description |
|---|---|
| User value | User có thể hỏi, chia sẻ, phản hồi và lưu nội dung hữu ích |
| Product value | Tạo lõi social interaction cho UEConnect |
| Business value | Biến app thành social platform thật, không chỉ là profile directory |
| Trust value | Chỉ verified users được đăng/comment |
| Safety value | Report/block/moderation gắn trực tiếp vào nội dung |
| UX value | Thảo luận rõ ràng, dễ đọc, dễ kiểm soát, không rối như comment section vô chủ |

### 1.4. Non-goals

Feature này không làm:

- Không hỗ trợ anonymous post/comment.
- Không hỗ trợ confession.
- Không hỗ trợ dating/crush/hot/match content.
- Không hỗ trợ comment thread vô hạn cấp.
- Không hỗ trợ livestream, story, reel.
- Không hỗ trợ marketplace listing.
- Không hỗ trợ job/recruiter post như LinkedIn.
- Không public bài viết ra internet.
- Không cho unverified/suspended/banned user tạo nội dung.
- Không cho user chỉnh sửa nội dung đã bị moderation lock nếu policy không cho phép.
- Không public report history hoặc moderation reason nhạy cảm cho người không có quyền.

## 4. Actors & Permissions

### 4.1. Actors

| Actor            |   Can Use? | Notes                                                                |
| ---------------- | ---------: | -------------------------------------------------------------------- |
| Guest            |         No | Redirect login                                                       |
| Registered user  |         No | Must verify                                                          |
| Pending user     |         No | Redirect verification status                                         |
| Verified Student |        Yes | Can post/comment/interact                                            |
| Verified Alumni  |        Yes | Can post/comment/interact                                            |
| Verified Advisor |        Yes | Can post/comment/interact                                            |
| Mentor           |        Yes | Can post/comment as verified user                                    |
| Club Manager     |        Yes | Can post/comment; community-specific permission handled in Community |
| Admin            |        Yes | Can create announcement/admin content if permitted                   |
| Moderator        |        Yes | Can moderate reported content                                        |
| Suspended user   | Limited/No | Cannot create/interact                                               |
| Banned user      |         No | Cannot access                                                        |

### 4.2. Required Permissions

| Action                            | Required Role        | Required Permission | Scope            |
| --------------------------------- | -------------------- | ------------------- | ---------------- |
| View post                         | verified active user | none                | visibility-based |
| Create post                       | verified active user | none                | own              |
| Edit own post                     | post owner           | none                | own              |
| Delete own post                   | post owner           | none                | own              |
| Create comment                    | verified active user | none                | visible post     |
| Edit own comment                  | comment owner        | none                | own              |
| Delete own comment                | comment owner        | none                | own              |
| React to post/comment             | verified active user | none                | visible content  |
| Save post                         | verified active user | none                | own              |
| Report post/comment               | verified active user | none                | own report       |
| Block author/commenter            | verified active user | none                | own safety       |
| Hide post/comment                 | moderator/admin      | `moderate_content`  | global/scoped    |
| Delete post/comment by moderation | moderator/admin      | `moderate_content`  | global/scoped    |
| Restore post/comment              | moderator/admin      | `moderate_content`  | global/scoped    |
| View report context               | moderator/admin      | `moderate_content`  | global/scoped    |

### 4.3. Access Rules

```txt id="vh94n6"
- Only active verified users can create posts/comments.
- User can edit/delete only their own posts/comments.
- Suspended users cannot create, edit, react, save, or comment.
- Banned users cannot access product content.
- Post/comment actions must recheck visibility and account status server-side.
- Blocked users cannot interact with each other's content.
- Hidden/deleted/removed content must not expose original body/media to normal users.
- Moderator/admin actions require permission and reason.
- Community posts must respect community membership and community status.
```

## 8. Post Rules

### 8.1. Post Content Rules

Recommended limits:

```txt id="1fy6by"
post body max length = 3000 characters
post body required unless media exists
image max count = 4
image max size = 5MB each
```

Post must have at least one:

```txt id="49bebv"
- body text
- image media
```

### 8.2. Post Visibility Rules

| Visibility         | Meaning                              | Priority |
| ------------------ | ------------------------------------ | -------- |
| `verified_users`   | Visible to verified UEConnect users  | P0       |
| `connections_only` | Visible only to accepted connections | P1       |
| `community_only`   | Visible in selected community        | P1       |
| `admin_targeted`   | Admin/system audience targeting      | P0/P1    |
| `draft`            | Not published                        | P0       |

Default:

```txt id="l0iryx"
verified_users
```

### 8.3. Post Status

| Status                  | Meaning                         |
| ----------------------- | ------------------------------- |
| `draft`                 | Not published                   |
| `active`                | Visible according to visibility |
| `edited`                | Active and edited               |
| `deleted_by_author`     | Soft deleted by owner           |
| `hidden_by_moderation`  | Hidden by moderator/admin       |
| `auto_hidden`           | Hidden by threshold/flag        |
| `removed_by_moderation` | Removed by moderator/admin      |
| `locked`                | Cannot receive new comments     |
| `archived`              | Future state                    |

### 8.4. Edit Post Rules

```txt id="u9165y"
- Author can edit own active post.
- Edited post should show edited indicator.
- Edit history is P2 but data model should be future-ready.
- Post under moderation review may be locked from editing depending on policy.
- Moderator/admin can still hide/delete after edit.
```

### 8.5. Delete Post Rules

```txt id="81y7zp"
- Author delete is soft delete.
- Deleted post no longer appears in feed.
- Post detail shows unavailable to normal users.
- Comments are hidden with parent post for normal users.
- Admin/moderator can still view for audit/moderation if permitted.
```

## 9. Comment Rules

### 9.1. Comment Content Rules

Recommended limits:

```txt id="j9pgsc"
comment max length = 1000 characters
reply max length = 1000 characters
```

Comment must have:

```txt id="wbkhpu"
body text
```

Comment image attachment:

```txt id="9015cj"
Out of scope for MVP unless explicitly enabled later.
```

### 9.2. Reply Depth

MVP supports:

```txt id="nle6sl"
one-level replies only
```

Meaning:

```txt id="txhtpf"
Post
├── Comment
│   ├── Reply
│   └── Reply
├── Comment
└── Comment
```

Not allowed:

```txt id="pkzd5t"
Reply to reply to reply to reply...
```

Vì đó là cách comment section trở thành mê cung, và không ai vào app trường để học môn khảo cổ UI.

### 9.3. Comment Status

| Status                  | Meaning                         |
| ----------------------- | ------------------------------- |
| `active`                | Visible                         |
| `edited`                | Active and edited               |
| `deleted_by_author`     | Soft deleted by owner           |
| `hidden_by_moderation`  | Hidden                          |
| `auto_hidden`           | Hidden by threshold/flag        |
| `removed_by_moderation` | Removed                         |
| `parent_deleted`        | Parent post/comment unavailable |
| `locked`                | Cannot be replied to            |

### 9.4. Delete Comment Rules

```txt id="9498kh"
- Author can soft delete own comment.
- If top-level comment is deleted, replies can be hidden or shown under placeholder depending on policy.
- MVP recommendation: show placeholder "Bình luận này không còn khả dụng" if replies exist.
- If no replies, comment can disappear from normal view.
```

### 9.5. Edit Comment Rules

```txt id="vgpglz"
- Author can edit own active comment.
- Edited comment should show edited indicator.
- Comment under moderation review may be locked from editing depending on policy.
```

## 10. Reaction & Save Rules

```txt id="4qd0z6"

support
```


```txt id="djqpcw"
like only
```

### 10.2. Reaction Rules

```txt id="w47rip"
- One active reaction per user per target.
- User can change reaction type.
- User can remove reaction.
- Suspended/banned users cannot react.
- User cannot react to hidden/deleted content.
```

Reaction targets:

```txt id="qpzr6a"
post
comment
```

### 10.3. Save Rules

```txt id="vb83wm"
- User can save visible posts.
- Save is private to user.
- Saved posts are removed/unavailable if post is deleted/hidden.
- User cannot save comments in MVP.
```

## 11. Main User Flow

### 11.1. Create Post Flow

```txt id="sczk30"
User opens composer
→ User enters text
→ User optionally attaches images
→ User selects visibility
→ System validates content/media/permission
→ System creates post
→ Feed updates
→ Analytics event recorded
```

### 11.2. Post Detail Flow

```txt id="cna9eu"
User opens post detail
→ System checks visibility/access
→ Load post, author, media, counts
→ Load comments paginated
→ Render post detail
→ User can comment/react/save/report/block if allowed
```

### 11.3. Create Comment Flow

```txt id="kbt59z"
User enters comment
→ System checks active account and visible post
→ System validates body
→ System creates comment
→ Comment count updates
→ Post author receives notification if applicable
```

### 11.4. Reply Flow

```txt id="8aeldt"
User taps reply on top-level comment
→ System checks comment is active and top-level
→ User enters reply
→ System validates body and depth
→ Reply is created
→ Parent comment author may receive notification
```

### 11.5. Report Flow

```txt id="v0tfqc"
User opens post/comment menu
→ Selects Report
→ Chooses reason
→ Optional description
→ System checks duplicate pending report
→ Report is created
→ Target enters moderation queue
→ Report action may auto-block target depending safety policy
→ UI shows confirmation
```

## 13. Alternative & Failure Flows

| Flow                           | Condition                                              | Expected Behavior |

| Body too long                  | Over 3000 chars                                        | Validation error  |

| Post hidden while viewing      | Show unavailable/placeholder                           |                   |
| Comment deleted before reply   | Reject reply or show parent unavailable                |                   |

| Duplicate report               | Show existing pending report state                     |                   |
| Notification fails             | Content action succeeds; notification retry            |                   |
| Analytics fails                | Content action succeeds; analytics retry/ignore safely |                   |

````
## 20. Security & Privacy

### 20.1. Security Rules

```txt id="ccf2gw"
- Server must authorize every create/edit/delete/react/save/report action.
- Post visibility must be enforced server-side.
- Comment creation must recheck parent post status.
- Reply creation must recheck parent comment depth/status.
- Media IDs must belong to current user before attaching.
- Suspended/banned users cannot create or interact.
- Blocked users cannot interact with each other's content.
- Hidden/deleted content must not leak body/media through API.
- Report details are visible only to reporter/admin/moderation.
````

### 20.2. Privacy Rules

```txt id="1j054t"
- Feed/post detail is verified-only.
- Public profile data in post card must be sanitized.
- Full MSSV must never be included.
- Email/phone/evidence/admin notes never appear in post/comment UI.
- Saved posts are private to saver.
- Analytics must not store raw post/comment body.
```

## 33. Final Feature Statement

```txt id="t0v7j3"
Post & Comment là interaction core của UEConnect Feed. Feature này cho phép verified active users tạo bài viết, bình luận, phản hồi một cấp, tương tác, lưu, báo cáo và kiểm soát nội dung của mình. Mọi hành động phải được authorize server-side, tôn trọng visibility/block/moderation/account status, không anonymous, không dating, không public internet feed và không leak dữ liệu nhạy cảm vào UI hoặc analytics.
```

## 34. Short Version

```txt id="gt1ls9"
Verified active mới được post/comment.
Post có text hoặc image.
Comment tối đa 1 cấp reply.
Không anonymous.
Không nested vô hạn.
Owner được xóa nội dung của mình.
Moderator được hide/delete/restore có reason.
Report luôn có.
Block thì chặn interaction.
Hidden/deleted thì không leak body/media.
Comment system mà không có moderation là mời drama vào ngồi ghế VIP.
```
