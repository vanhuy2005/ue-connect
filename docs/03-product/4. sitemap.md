# Sitemap

## 1. Purpose

File này định nghĩa toàn bộ **sitemap** của UEConnect PWA.

Mục tiêu:

- Mô tả đầy đủ toàn bộ page chính của sản phẩm.
- Map page với route, actor, access rule, feature spec, priority và layout.
- Làm nền cho `page-specs`, routing trong Laravel, permission middleware, navigation design, mobile layout và QA test.
- Tách rõ public pages, auth pages, verification pages, user app pages, admin pages, error/state pages và future-ready pages.
- Giúp team hình dung toàn bộ PWA như một sản phẩm thật, không phải một đống screen rời rạc được nối bằng niềm tin và `href="#"`.

Sitemap trả lời:

```txt
PWA có những page nào?
Route URL là gì?
Ai được truy cập?
Page đó thuộc module nào?
Page đó nằm ở navigation nào?
Page đó cần feature spec nào?
Page đó P0/P1/P2/P3?
Page đó có trạng thái loading/empty/error/permission không?
````

---

## 2. Sitemap Principles

UEConnect dùng các nguyên tắc sitemap sau:

```txt
01. Route phải phản ánh product domain.
02. Page public, app và admin phải tách rõ.
03. Verified-only page phải có route guard.
04. Admin page phải có permission guard.
05. Mobile navigation phải ưu tiên 5 tab chính.
06. Page không có business value thì không thêm.
07. Page future-ready phải đánh dấu rõ, không giả vờ là MVP.
08. Error/empty/loading/state pages phải được tính từ đầu.
09. Page nào chứa sensitive data phải ghi rõ access rule.
10. Sitemap phải map được sang feature spec và page spec.
```

---

## 3. Route Group Overview

UEConnect PWA chia thành 7 route groups chính:

```txt
01. Public routes
02. Authentication routes
03. Verification routes
04. App routes
05. Admin routes
06. Support / policy routes
07. System / error / state routes
```

---

## 4. Route Group Summary

| Route Group      | Prefix                | Access                               | Purpose                              |
| ---------------- | --------------------- | ------------------------------------ | ------------------------------------ |
| Public           | `/`                   | Guest + all users                    | Landing, product intro, help         |
| Auth             | `/auth`               | Guest / unauthenticated              | Login, register, password            |
| Verification     | `/verification`       | Authenticated but not fully verified | Submit and track verification        |
| App              | `/app`                | Active verified users                | Main PWA user experience             |
| Admin            | `/admin`              | Admin + permission                   | Operations, moderation, verification |
| Support / Policy | `/support`, `/policy` | Depends on page                      | Help, safety, guidelines             |
| System / State   | `/system`, errors     | Contextual                           | Error, offline, permission denied    |

---

## 5. Access Level Legend

| Access Level     | Meaning                                   |
| ---------------- | ----------------------------------------- |
| Public           | Anyone can access                         |
| Guest only       | Only unauthenticated users                |
| Authenticated    | Logged in user                            |
| Unverified       | Logged in but not verified                |
| Verified         | Active verified user                      |
| Student          | Verified user with `student` primary role |
| Alumni           | Verified user with `alumni` primary role  |
| Advisor          | Verified user with `advisor` primary role |
| Admin            | User with `admin` primary role            |
| Permission-based | Requires specific permission              |
| System           | Internal or error route                   |

# 34. Recommended Laravel Route Group Direction

Conceptual route structure:

```php
// Public
Route::get('/', LandingController::class);

// Auth
Route::prefix('auth')->group(function () {
    Route::get('/login', ...);
    Route::get('/register', ...);
});

// Verification
Route::middleware(['auth'])->prefix('verification')->group(function () {
    Route::get('/start', ...);
    Route::get('/student', ...);
    Route::get('/status', ...);
});

// App
Route::middleware(['auth', 'active', 'verified'])->prefix('app')->group(function () {
    Route::get('/home', ...);
    Route::get('/discovery', ...);
    Route::get('/messages', ...);
    Route::get('/mentors', ...);
    Route::get('/communities', ...);
    Route::get('/profile', ...);
    Route::get('/settings', ...);
});

// Admin
Route::middleware(['auth', 'active', 'admin'])->prefix('admin')->group(function () {
    Route::get('/', ...)->middleware('can:view-admin-dashboard');
    Route::get('/verification', ...)->middleware('can:review-verification');
    Route::get('/moderation', ...)->middleware('can:moderate-content');
    Route::get('/users', ...)->middleware('can:manage-users');
    Route::get('/permissions', ...)->middleware('can:manage-permissions');
    Route::get('/audit', ...)->middleware('can:view-audit-log');
});
```

Đây chỉ là direction, không phải final implementation. Đừng copy nguyên rồi gọi là architecture, dù ngành này vẫn làm vậy đều đặn.
