# empty-states.md

## 1. Purpose
Describe how the UI behaves when content is missing or a collection is empty.

## 2. Design Decision
Use friendly guidance, a clear next step, and a visually calm layout.

## 3. Rationale
Empty states should help users recover, not make the product feel broken.

## 4. Rules
- Explain why the state is empty when needed.
- Offer one clear next action.
- Keep illustrations or icons lightweight.

## 5. Do / Don't
Do: make the next step obvious.

Don't: use a dead-end empty screen.

## 6. Tokens / Specs
Use centered content, moderate spacing, and a supportive CTA.

## 7. Component / Screen Impact
Impacts list views, messages, search results, and onboarding gaps.

## 8. QA Checklist
- Empty state has a clear CTA or recovery path.
- Tone stays supportive.
- The layout is not visually heavy.

## 9. AI Prompt Notes
Ask for a calm, useful empty state with one meaningful next action.
