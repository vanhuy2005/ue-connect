# profile.md

## 1. Purpose
Describe the profile experience for UEers, mentors, alumni, clubs, and other roles.

## 2. Design Decision
Make the profile visually rich but structured, with identity, trust, and action surfaces clearly separated.

## 3. Rationale
Profiles are a trust object, so the layout must help users understand who someone is at a glance.

## 4. Rules
- Keep the avatar and identity block prominent.
- Surface verification or role badges carefully.
- Make primary actions easy to find.

## 5. Do / Don't
Do: balance personality and clarity.

Don't: stack too much text directly over busy imagery.

## 6. Tokens / Specs
Use rounded cards, strong overlay when text sits on image, and clear metadata spacing.

## 7. Component / Screen Impact
Impacts profile card, cover area, avatar, badges, actions, and bio sections.

## 8. QA Checklist
- Identity is clear above the fold.
- Verification or role cues are visible.
- The page still works without a cover image.

## 9. AI Prompt Notes
Ask for a profile screen with trust-first hierarchy and social warmth.
