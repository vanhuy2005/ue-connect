# settings.md

## 1. Purpose
Describe the account, privacy, and app configuration experience.

## 2. Design Decision
Use a simple, grouped settings layout with obvious sections and safe defaults.

## 3. Rationale
Settings should help users act quickly, not force them to decode the app.

## 4. Rules
- Group related settings together.
- Make destructive actions easy to recognize.
- Keep privacy and safety settings easy to reach.

## 5. Do / Don't
Do: use clean section labels and stable patterns.

Don't: hide important settings behind ambiguous labels.

## 6. Tokens / Specs
Use list rows, toggles, secondary helper text, and clear divider spacing.

## 7. Component / Screen Impact
Impacts sections, toggles, rows, danger actions, and account options.

## 8. QA Checklist
- Categories are easy to understand.
- Critical actions are clearly labeled.
- Settings remain readable on mobile.

## 9. AI Prompt Notes
Ask for a calm settings page with strong hierarchy and low visual noise.
