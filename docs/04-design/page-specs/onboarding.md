# onboarding.md

## 1. Purpose
Describe the first-run experience that introduces UEConnect and builds trust fast.

## 2. Design Decision
Use a brand-led, low-friction onboarding flow with a primary CTA and minimal cognitive load.

## 3. Rationale
This is the first trust moment, so it must explain the product quickly without overwhelming new users.

## 4. Rules
- Start with the brand anchor and product promise.
- Keep steps short and mobile-friendly.
- Reinforce verified HCMUE identity early.

## 5. Do / Don't
Do: keep one clear primary CTA per screen.

Don't: overload onboarding with too many feature explanations.

## 6. Tokens / Specs
Use the primary blue-only gradient, pill CTA, full-width mobile buttons, and large spacing around the hero message.

## 7. Component / Screen Impact
Impacts splash, welcome, sign-up, permission gating, and intro slides.

## 8. QA Checklist
- The value proposition is clear in one glance.
- The CTA is easy to find on mobile.
- The screen does not feel like a form wall.

## 9. AI Prompt Notes
Ask for a verified HCMUE social onboarding flow with a premium blue gradient and calm spacing.
