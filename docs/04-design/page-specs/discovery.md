# discovery.md

## 1. Purpose
Describe the browsing and search experience for discovering people, mentors, clubs, and events.

## 2. Design Decision
Use filterable discovery modules with clear segmentation and strong search affordances.

## 3. Rationale
Discovery must support both intentional search and casual browsing without confusion.

## 4. Rules
- Filters must be easy to reset.
- Discovery categories must be visually distinct.
- Search results need trust cues and fast feedback.

## 5. Do / Don't
Do: keep browsing flexible.

Don't: make discovery feel like a cluttered filter dump.

## 6. Tokens / Specs
Use search input, chips, cards, badges, and grid spacing that supports rapid scanning.

## 7. Component / Screen Impact
Impacts search, filters, result cards, empty states, and saved discovery shortcuts.

## 8. QA Checklist
- Filters are understandable and reversible.
- Discovery results are grouped logically.
- Search feedback is immediate.

## 9. AI Prompt Notes
Ask for a discovery surface that feels curated, not algorithmically chaotic.
