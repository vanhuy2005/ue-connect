# Product Scope

## 1. Purpose

File này định nghĩa phạm vi sản phẩm của UEConnect.

Mục tiêu:

- Làm rõ UEConnect sẽ làm gì.
- Làm rõ UEConnect không làm gì.
- Chốt phạm vi MVP.
- Chốt các feature cần chuẩn bị kiến trúc nhưng chưa bắt buộc hoàn thiện ngay.
- Chốt ranh giới giữa product scope, future scope và out-of-scope.
- Giúp business, requirements, product, design, database và system architecture không tự đá nhau.

Product scope là hàng rào bảo vệ project khỏi việc “thêm nhẹ một tính năng nữa”. Đây là câu nói đã giết rất nhiều deadline, nhiều repo, và có lẽ cả vài giấc ngủ của sinh viên công nghệ thông tin.

## 2. Product Scope Summary

UEConnect là một **verified student social platform** dành riêng cho HCMUE.

Phạm vi chính của sản phẩm:

```txt
Verified identity
Social feed
Discovery profile
Greeting / connection
Messaging 1:1
Mentor / advisor / alumni connection
Community / CLB
Notifications
Privacy / settings
Report / block
Admin verification
Admin moderation
Audit
PWA-first web app
````

UEConnect không phải:

```txt
Dating app
Super app sinh viên
Multi-university platform
Student portal
Job board
Payment system
Marketplace
Anonymous confession app
Native mobile app ở giai đoạn đầu
```

## 3. Scope Principles

UEConnect tuân theo các nguyên tắc scope sau.

## 3.1. HCMUE-only

UEConnect chỉ dành cho cộng đồng HCMUE.

Hệ thống không thiết kế cho multi-university trong giai đoạn hiện tại.

Điều này ảnh hưởng đến:

- Brand identity.
- Verification.
- Role model.
- Community/CLB.
- Admin operation.
- Content policy.
- Data model.
- Product language.

## 3.2. Verified-first

User cần được xác thực để tham gia đầy đủ.

Trust không phải feature phụ. Trust là điều kiện tồn tại của sản phẩm.

Verified identity dựa trên:

- MSSV uniqueness.
- Upload minh chứng.
- Email trường `hcmue.edu.vn` nếu có.
- Admin review.
- Role xác thực.
- Audit.

## 3.3. Student-first

Sinh viên HCMUE là user trung tâm.

Các role khác như alumni, advisor, admin, club manager tồn tại để hỗ trợ hệ sinh thái sinh viên, không làm lệch trọng tâm sản phẩm.

## 3.4. Social without dating

UEConnect hỗ trợ kết nối và làm quen, nhưng không phải dating app.

Không dùng:

```txt
match
swipe
crush
hot
dating
ghép đôi
tán tỉnh
```

Dùng:

```txt
UEer
gửi lời chào
kết nối
bạn cùng khoa
cùng học
mentor
cộng đồng
CLB
```

## 3.5. Enterprise planning from day one

Dù là portfolio case study, UEConnect vẫn được planning theo hướng có thể triển khai thật.

Do đó, scope phải bao gồm:

- Permission.
- Moderation.
- Audit.
- Verification.
- Privacy.
- Safety.
- Admin operation.
- Traceability.

## 4. Target Platform Scope

## 4.1. Primary Platform

UEConnect là:

```txt
PWA-first web application
```

Stack định hướng:

```txt
Laravel
Blade
TailwindCSS
Vite
MS SQL Server
CSS Modules nếu cần
PWA
```

## 4.2. PWA Scope

PWA nằm trong scope.

PWA cần hỗ trợ:

- Responsive web app.
- Mobile-first layout.
- Installable app shell.
- Manifest.
- App icon.
- Basic offline fallback.
- App-like navigation.
- Future-ready notification direction.

PWA không đồng nghĩa với native app.

Không nằm trong scope hiện tại:

- iOS native app.
- Android native app.
- React Native app.
- Flutter app.
- App Store / Google Play release.

## 4.3. API / Mobile Readiness

UEConnect chưa cần native mobile app ở MVP, nhưng backend nên được thiết kế có khả năng mở rộng API sau này.

Scope hiện tại:

```txt
Web app first
PWA first
API-ready architecture where reasonable
```

Không cần:

```txt
Full public API
Third-party developer API
Mobile-native API contract hoàn chỉnh
```

## 6. Role Scope

## 6.1. Primary System Roles

Mỗi user có một role hệ thống chính:

```txt
Student
Alumni
Advisor
Admin
```

Rules:

- User không tự ý đổi role chính.
- Role chính được xác định thông qua verification.
- Student có thể chuyển thành Alumni theo cơ chế năm học/tốt nghiệp hoặc admin action.
- Admin là role quản trị đặc biệt.

## 6.2. Admin-granted Permissions

Ngoài role chính, admin có thể cấp quyền phụ:

```txt
Club Manager
Community Moderator
Mentor Permission
Verification Reviewer
Content Moderator
System Admin Permission
```

Role chính thể hiện **identity**.

Permission phụ thể hiện **quyền thao tác**.

## 6.3. Out-of-scope Role Complexity

Không làm ở giai đoạn hiện tại:

- Role hierarchy quá sâu.
- RBAC phức tạp kiểu enterprise SaaS đầy đủ.
- Organization-level tenant.
- Multi-school role.
- User tự request mọi permission trong UI công khai.
- Role marketplace hoặc public badge system.

## 8. MVP Scope

Trong tư duy hiện tại của dự án, MVP là một bản P0 khá đầy đủ, không phải prototype rỗng.

## 8.1. MVP P0 Modules

Các module P0:

```txt
Auth + mã sinh viên verification
Profile setup
Home feed
Create post
Comment
Discovery UEers
Gửi lời chào / connection
Messaging 1:1
Mentor
Community / CLB
Notifications
Settings / privacy
Report / block
Admin verification
Admin moderation
```

## 8.2. MVP Success Expectation

MVP phải chứng minh được:

- User có thể đăng ký và được xác thực.
- MSSV không thể bị dùng bởi nhiều tài khoản.
- User có thể hoàn thiện profile.
- User có thể đọc và đăng bài.
- User có thể comment.
- User có thể khám phá UEers.
- User có thể gửi lời chào.
- User có thể nhắn tin 1:1 khi đủ điều kiện.
- User có thể gửi mentor request.
- User có thể tham gia hoặc xem community/CLB.
- User có thể nhận notification.
- User có thể quản lý privacy cơ bản.
- User có thể report/block.
- Admin có thể duyệt tài khoản.
- Admin có thể xử lý report.
- Admin action có audit.

## 8.3. MVP Quality Bar

MVP phải đạt:

```txt
Functional enough for real deployment planning
Documented enough for portfolio
Secure enough for verified student environment
Moderated enough for campus social product
Responsive enough for PWA-first usage
Structured enough for future expansion
```

MVP không được là:

```txt
Static mockup
Landing page giả app
Feed không có verification
Social app không có moderation
Chat app không có permission
Community không có admin approval
Portfolio chỉ có screenshot không có reasoning
```

## 9. Module Scope Details

## 9.1. Public Landing

### In Scope

Landing page tối giản cho Guest.

Chức năng:

- Giới thiệu UEConnect.
- Giải thích đây là nền tảng dành cho HCMUE.
- Nêu giá trị chính: kết nối, học cùng, mentor, cộng đồng.
- Giải thích cần verified account.
- CTA đăng ký.
- CTA đăng nhập.
- Link guideline/privacy cơ bản.

### Out of Scope

- Landing marketing quá dài.
- Blog marketing.
- SEO content hub.
- Public feed.
- Public profile.
- Public community browsing đầy đủ.

Landing page chỉ để user hiểu sản phẩm và bắt đầu đăng ký. Không biến nó thành brochure trường có animation. Nhân loại đã chịu đủ landing page dài 15 section rồi.

## 9.2. Authentication

### In Scope

- Register.
- Login.
- Logout.
- Forgot password.
- Reset password.
- Session handling.
- Account status redirect.
- Basic security validation.

### Out of Scope

- Social login Facebook/Google ở MVP.
- SSO chính thức với hệ thống trường nếu chưa có hạ tầng.
- Multi-factor authentication nâng cao.
- Enterprise SAML/OIDC.

Authentication nên được xây trên Laravel best practices. Nếu sau này cần API/PWA nâng cao, có thể chuẩn bị direction với Sanctum.

## 9.3. Verification

### In Scope

Verification là core scope.

Bao gồm:

- MSSV input.
- MSSV uniqueness check.
- Upload minh chứng.
- Email `hcmue.edu.vn` nếu có.
- Admin review.
- Account status: pending, approved, rejected, need more information.
- Re-submit evidence.
- Verification audit.
- Evidence privacy.

Minh chứng hợp lệ:

```txt
Bảng điểm
Giấy nhập học
Bằng tốt nghiệp
Thẻ sinh viên
Email trường
Tài liệu pháp lý số chứng minh danh tính liên quan HCMUE
```

### Out of Scope

- Tự động đồng bộ database chính thức của trường nếu chưa có quyền.
- OCR tự động xác thực giấy tờ ở MVP.
- AI verification decision tự động.
- Public display MSSV đầy đủ.

### Critical Rule

```txt
Một MSSV chỉ được liên kết với một tài khoản duy nhất.
```

Rule này phải có ở:

- Requirement.
- Database constraint.
- Admin review.
- Edge case.
- Test case.
- Audit log.

## 10. Future-ready Architecture Scope

Các feature sau **chưa bắt buộc hoàn thiện đầy đủ trong MVP**, nhưng kiến trúc phải chuẩn bị.

## 10.1. Community Chat

Chuẩn bị:

- Conversation/channel model.
- Community permission.
- Message moderation.

Không ảnh hưởng core metrics ban đầu.
