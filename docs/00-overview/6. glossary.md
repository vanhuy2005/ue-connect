# Glossary

## 1. Purpose

File này định nghĩa các thuật ngữ chính thức được sử dụng trong UEConnect.

Mục tiêu:

- Chuẩn hóa cách gọi thuật ngữ trong toàn bộ docs.
- Giảm mâu thuẫn giữa product, business, requirements, design và engineering.
- Làm rõ thuật ngữ nào được phép dùng trong UI.
- Làm rõ thuật ngữ nào chỉ dùng nội bộ.
- Làm rõ thuật ngữ nào bị cấm vì làm lệch product identity.
- Hỗ trợ viết use case, user flow, page spec, database, component, content tone và admin policy nhất quán.

Glossary không phải phụ lục cho vui. Nó là thứ giữ cho cả repo không biến thành một nồi lẩu từ vựng, nơi `connection`, `friend`, `match`, `greeting`, `follow` bị ném chung rồi gọi là “tạm hiểu nha”. Không, ta không tạm hiểu. Ta định nghĩa.

---

## 2. Naming Principles

UEConnect dùng thuật ngữ theo các nguyên tắc sau:

```txt
Student-first
HCMUE-rooted
Verified-first
Social but not dating
Human but not childish
Growth-oriented but not corporate
Enterprise-ready but easy to understand
````

Thuật ngữ phải:

- Dễ hiểu với sinh viên HCMUE.
- Nhất quán giữa UI và docs.
- Không kéo sản phẩm về dating vibe.
- Không làm sản phẩm giống portal hành chính.
- Không quá technical trong UI.
- Có thể map được sang database, requirement và page spec.

## 3. Language Rules

## 3.1. UI Language

UI nên dùng tiếng Việt dễ hiểu, nhưng giữ một số thuật ngữ tiếng Anh nếu đó là thuật ngữ product/technical quen thuộc.

Ví dụ được dùng:

```txt
UEer
Verified UEer
Mentor
Profile
Community
Feed
Admin
Moderator
Permission
Audit log
```

Ví dụ nên dùng tiếng Việt:

```txt
Gửi lời chào
Kết nối
Bạn cùng khoa
Cùng học
Đang chờ duyệt
Cần bổ sung thông tin
Báo cáo
Chặn
```

## 3.2. Documentation Language

Docs viết bằng tiếng Việt, nhưng thuật ngữ chuyên môn giữ nguyên tiếng Anh khi cần:

```txt
Product scope
Business rules
Functional requirements
Non-functional requirements
Role-permission matrix
Traceability matrix
Audit log
Moderation queue
User flow
Page spec
Design token
```

## 3.3. Forbidden Language

Không dùng các từ sau trong product UI, docs user-facing hoặc feature naming:

```txt
crush
match
swipe
hot
dating
ghép đôi
tán tỉnh
người yêu
hẹn hò
đối tượng phù hợp
```

Lý do:

```txt
Các từ này kéo UEConnect về dating app, làm giảm trust trong môi trường trường học.
```

Nếu cần mô tả interaction học từ Tinder trong docs nội bộ, ghi rõ:

```txt
Discovery interaction inspired by Tinder pattern, but not dating language or dating vibe.
```

## 4. Product Identity Terms

## 4.1. UEConnect

### Definition

```txt
UEConnect là verified student social platform dành riêng cho HCMUE.
```

### Usage

Dùng trong:

- Product name.
- Logo.
- Landing page.
- Docs.
- Admin UI.
- System notification.

### Do

```txt
UEConnect giúp bạn kết nối với UEers cùng khoa, cùng học và cùng mục tiêu.
```

### Do Not

```txt
UE Connect
UEConnector
UE Dating
HCMUE Tinder
```

### Notes

Tên chính thức là:

```txt
UEConnect
```

Không thêm dấu cách nếu đang nói về brand/product name.

---

## 4.2. HCMUE

### Definition

```txt
HCMUE là phạm vi trường mà UEConnect phục vụ.
```

### Usage

Dùng để chỉ bối cảnh chính thức của sản phẩm:

- HCMUE community.
- HCMUE student.
- HCMUE context.
- HCMUE identity.

### Notes

UEConnect hiện chỉ dành cho HCMUE, không phải multi-university platform.

---

## 4.3. UEer

### Definition

```txt
UEer là cách gọi thân thiện cho thành viên thuộc cộng đồng UEConnect/HCMUE.
```

### Usage

Dùng trong UI social:

```txt
Khám phá UEers
Gửi lời chào đến UEer này
UEers cùng khoa
UEers có cùng mối quan tâm
```

### Meaning

UEer thể hiện:

- Thành viên thuộc cộng đồng HCMUE.
- Có identity trong UEConnect.
- Có thể là student, alumni hoặc advisor tùy context.

### Do

```txt
Khám phá UEers cùng khoa với bạn.
```

### Do Not

```txt
User hot gần bạn
Đối tượng phù hợp
Match mới
```

---

## 4.4. Verified UEer

### Definition

```txt
Verified UEer là user đã được xác thực danh tính hợp lệ trong hệ thống UEConnect.
```

### Verification Sources

Có thể xác thực bằng:

- MSSV.
- Upload minh chứng.
- Email hcmue.edu.vn nếu có.
- Admin review.
- Evidence phù hợp với role.

### Usage

Dùng trong:

- Profile.
- Post author row.
- Discovery card.
- Messaging header.
- Mentor profile.
- Admin verification.

### UI Copy

Good:

```txt
Đã xác thực UEer
Verified UEer
Tài khoản đã được xác thực
```

Bad:

```txt
Tài khoản thật 100%
An toàn tuyệt đối
Không thể giả mạo
```

### Notes

Verified không có nghĩa là hệ thống đảm bảo tuyệt đối mọi hành vi của user. Verified chỉ nghĩa là identity đã qua quy trình xác thực.

## 5. User & Role Terms

## 5.1. User

### Definition

```txt
User là bất kỳ tài khoản nào trong hệ thống UEConnect.
```

### Internal Usage

Dùng trong:

- Database.
- Requirement.
- Backend.
- Admin.
- Audit.

### UI Usage

Trong UI user-facing, ưu tiên dùng thuật ngữ cụ thể hơn:

```txt
UEer
Sinh viên
Alumni
Mentor
Admin
```

Không nên dùng chữ `user` quá nhiều trong UI tiếng Việt.

---

## 5.2. Guest

### Definition

```txt
Guest là người chưa đăng nhập hoặc chưa có tài khoản UEConnect.
```

### Capabilities

Guest có thể:

- Xem landing page.
- Đăng ký.
- Đăng nhập.
- Xem thông tin giới thiệu.
- Xem guideline/privacy cơ bản nếu public.

Guest không thể:

- Đăng bài.
- Comment.
- Gửi lời chào.
- Nhắn tin.
- Xem discovery đầy đủ.
- Gửi mentor request.
- Tham gia community.

---

## 5.3. Student

### Definition

```txt
Student là sinh viên HCMUE đang học, có primary system role là Student.
```

### Verification

Student cần xác thực bằng:

- MSSV.
- Minh chứng sinh viên.
- Email trường nếu có.
- Admin review.

### Main Features

Student có thể dùng:

- Feed.
- Post/comment.
- Discovery.
- Greeting.
- Messaging.
- Mentor request.
- Community.
- Report/block.

---

## 5.4. Alumni

### Definition

```txt
Alumni là cựu sinh viên HCMUE đã được xác thực.
```

### Verification

Alumni có thể xác thực bằng:

- Bằng tốt nghiệp.
- Minh chứng từng học tại HCMUE.
- MSSV nếu còn.
- Email trường nếu có.
- Admin review.

### Main Features

Alumni có thể:

- Tạo alumni profile.
- Đăng bài chia sẻ kinh nghiệm.
- Tham gia community.
- Trở thành mentor nếu được cấp permission.
- Nhắn tin theo permission model.

### Notes

Không viết sai thành:

```txt
Alumina
```

Tên đúng là:

```txt
Alumni
```

Vâng, chữ này đáng ghét, nhưng ta vẫn phải viết đúng.

---

## 5.5. Advisor

### Definition

```txt
Advisor là cố vấn học tập, giảng viên hoặc người hỗ trợ sinh viên được xác thực/cấp quyền trong UEConnect.
```

### Role

Advisor có thể:

- Hỗ trợ sinh viên.
- Nhận mentor request.
- Chia sẻ tài nguyên.
- Tham gia mentor/community flow.

### Notes

Advisor có thể liên quan mentor, nhưng không phải mọi advisor đều phải là mentor active.

---

## 5.6. Mentor

### Definition

```txt
Mentor là user có khả năng hỗ trợ sinh viên thông qua mentor request.
```

### Important Distinction

Mentor có thể là:

- Alumni.
- Advisor.
- Senior student nếu được cấp permission.
- Người được admin xác thực/cấp quyền.

Mentor không nhất thiết là primary system role.

### Recommended Model

```txt
Mentor nên là permission/capability được admin grant, không phải primary system role riêng trong giai đoạn hiện tại.
```

### UI Usage

Good:

```txt
Tìm mentor
Gửi mentor request
Mentor có thể hỗ trợ bạn về chủ đề này
```

Bad:

```txt
Kết nối career chuyên nghiệp
Ứng tuyển mentor
Profile tuyển dụng
```

---

## 5.7. Admin

### Definition

```txt
Admin là user có quyền quản trị hệ thống UEConnect.
```

### Responsibilities

Admin có thể:

- Duyệt tài khoản.
- Xem verification evidence.
- Approve/reject verification.
- Request more information.
- Quản lý user.
- Gán role/permission.
- Xử lý report.
- Kiểm duyệt nội dung.
- Quản lý community.
- Xem audit log.

### Notes

Admin action quan trọng phải có audit.

---

## 5.8. Moderator

### Definition

```txt
Moderator là user hoặc admin permission chịu trách nhiệm kiểm duyệt nội dung.
```

### Difference from Admin

Admin có quyền rộng hơn.

Moderator có thể chỉ được cấp một số quyền:

- Xem moderation queue.
- Xử lý report.
- Hide/delete content trong phạm vi.
- Warn user nếu được phép.

### Notes

Moderator có thể là permission phụ do Admin cấp.

---

## 5.9. Club Manager

### Definition

```txt
Club Manager là user được admin cấp permission để quản lý một CLB/cộng đồng cụ thể.
```

### Important Distinction

Club Manager không phải primary system role.

Club Manager là:

```txt
admin-granted permission / assignment
```

### Responsibilities

Club Manager có thể:

- Quản lý community được giao.
- Đăng thông báo.
- Quản lý thành viên nếu được cấp quyền.
- Kiểm duyệt nội dung community nếu được cấp quyền.
- Đề xuất tài nguyên/sự kiện nếu feature có.

---

## 6. Role & Permission Terms

## 6.1. Primary System Role

### Definition

```txt
Primary system role là role chính thể hiện identity của user trong UEConnect.
```

### Allowed Values

```txt
Student
Alumni
Advisor
Admin
```

### Rules

- Mỗi user chỉ có một primary system role.
- User không tự ý đổi primary role.
- Role được xác định qua verification.
- Student có thể chuyển thành Alumni theo rule rõ.
- Admin là role đặc biệt.

---

## 6.2. Permission

### Definition

```txt
Permission là quyền thao tác cụ thể được hệ thống hoặc admin cấp cho user.
```

### Examples

```txt
manage_club
moderate_content
review_verification
view_audit_log
manage_roles
mentor_access
```

### Notes

Permission khác primary role.

Một user có role `Student` vẫn có thể có permission `manage_club` nếu được admin cấp làm Club Manager.

---

## 6.3. Admin-granted Permission

### Definition

```txt
Admin-granted permission là quyền phụ do admin cấp cho user.
```

### Examples

- Club Manager.
- Community Moderator.
- Mentor Permission.
- Verification Reviewer.
- Content Moderator.

### Audit Requirement

Mọi thay đổi permission quan trọng phải ghi audit log.

---

## 6.4. Role Transition

### Definition

```txt
Role transition là quá trình chuyển primary role của user từ trạng thái này sang trạng thái khác.
```

### Main Case

```txt
Student → Alumni
```

### Trigger Options

Có thể dựa trên:

- Năm học.
- Ngày tốt nghiệp.
- Admin action.
- User request + evidence.
- Policy cụ thể.

### Notes

Role transition cần audit.

---

## 7. Verification Terms

## 7.1. Verification

### Definition

```txt
Verification là quy trình xác thực user thuộc cộng đồng HCMUE và có role phù hợp.
```

### Includes

- MSSV.
- Evidence upload.
- Email hcmue.edu.vn nếu có.
- Admin review.
- Status update.
- Audit log.

---

## 7.2. MSSV / Student ID

### Definition

```txt
MSSV là mã số sinh viên, dùng làm identity anchor chính cho Student.
```

### Rules

- Một MSSV chỉ được liên kết với một tài khoản.
- MSSV không được public đầy đủ nếu không cần.
- MSSV duplicate phải có flow xử lý.
- MSSV cần được audit nếu admin thay đổi/liên kết lại.

### UI Usage

Good:

```txt
Mã sinh viên
```

Internal:

```txt
student_id
mssv
```

---

## 7.3. Evidence

### Definition

```txt
Evidence là minh chứng user upload để xác thực danh tính hoặc liên hệ với HCMUE.
```

### Examples

- Bảng điểm.
- Giấy nhập học.
- Bằng tốt nghiệp.
- Thẻ sinh viên.
- Email trường.
- Tài liệu pháp lý số.

### Rules

- Evidence là dữ liệu nhạy cảm.
- Không public.
- Chỉ admin có permission mới xem.
- Access nên có audit.
- Cần retention policy.

### UI Copy

Good:

```txt
Minh chứng xác thực
Tải lên minh chứng
Thông tin này chỉ dùng để xác thực tài khoản
```

Bad:

```txt
Upload giấy tờ cho vui
Gửi đại cái gì cũng được
```

---

## 7.4. Verification Status

### Definition

```txt
Verification status là trạng thái xét duyệt tài khoản.
```

### Allowed Values

```txt
Draft
Pending
Approved
Rejected
Need more information
Suspended
```

### UI Labels

| Internal                | UI Label              |
| ----------------------- | --------------------- |
| `draft`                 | Chưa gửi xác thực     |
| `pending`               | Đang chờ duyệt        |
| `approved`              | Đã xác thực           |
| `rejected`              | Bị từ chối            |
| `need_more_information` | Cần bổ sung thông tin |
| `suspended`             | Tài khoản bị tạm khóa |

---

## 7.5. Account Status

### Definition

```txt
Account status là trạng thái sử dụng tài khoản trong hệ thống.
```

### Examples

```txt
Active
Pending verification
Rejected verification
Suspended
Banned
Deactivated
Deleted
```

### Difference from Verification Status

Verification status chỉ nói về quy trình xác thực.

Account status nói về khả năng tài khoản được sử dụng trong hệ thống.

Một tài khoản có thể verified nhưng bị suspended vì vi phạm policy.

## 8. Social Product Terms

```txt
Feed là dòng nội dung social chính của UEConnect.
```

- Delete.

## 8.5. Save

```txt
Save là hành động lưu post hoặc profile để xem lại sau.
```

### Notes

## 9. Discovery & Connection Terms

## 9.1. Discovery

### Definition

```txt
Discovery là module giúp user khám phá UEers khác dựa trên ngữ cảnh học tập, cộng đồng và mối quan tâm.
```

### Purpose

Discovery giúp user tìm:

- Bạn cùng khoa.
- Bạn cùng khóa.
- Bạn cùng môn.
- Bạn cùng mục tiêu học tập.
- Mentor/alumni/advisor phù hợp.
- UEers có cùng interest.

### Not Dating

Discovery không phải dating flow.

Không dùng:

```txt
match
swipe
crush
hot
dating
```

---

## 9.2. Discovery Profile Card

### Definition

```txt
Discovery Profile Card là card hiển thị thông tin tóm tắt của một UEer trong Discovery.
```

### Should Include

- Avatar.
- Display name.
- Verified badge.
- Faculty/major/cohort.
- Bio ngắn.
- Shared context.
- Interests.
- CTA gửi lời chào.

---

## 9.3. Greeting / Gửi lời chào

### Definition

```txt
Greeting là hành động user gửi lời chào đến một UEer để bắt đầu kết nối.
```

### UI Label

```txt
Gửi lời chào
```

### Purpose

Greeting thay thế cho các khái niệm dating như `match`.

### States

```txt

Sent
Pending
Accepted
Declined

Blocked
```

### Notes

Greeting có thể kèm message ngắn.

---

## 9.4. Connection

### Definition

```txt
Connection là quan hệ giữa hai user sau khi lời chào được chấp nhận hoặc sau một interaction được hệ thống cho phép.
```

### Usage

Connection có thể mở quyền:

- Messaging 1:1.
- Follow-up interaction.
- Context visibility.

### Not Equal To

Connection không phải:

- Friend theo Facebook.
- Match theo Tinder.
- Follow theo Instagram.
- Contact theo danh bạ.

```txt
Connection state là trạng thái quan hệ giữa hai user.
```

### Allowed Values

```txt
Declined
Blocked
```

## 10.4. Messaging Permission

### Definition

```txt
Messaging permission là điều kiện cho phép user nhắn tin với user khác.
```

## 22. Naming Consistency Rules

## 22.1. Product Name

Always:

```txt
UEConnect
```

Never:

```txt
UE Connect
UE connector
UE-Connect
```

Trừ khi tên file hoặc route cần kebab-case:

```txt
ue-connect
```

## 22.2. User-facing Identity

Use:

```txt
UEer
Verified UEer
```

Avoid:

```txt
member
user
account holder
```

trong UI social.

## 22.3. Greeting

Use:

```txt
Gửi lời chào
```

Internal:

```txt
greeting
```

Do not use:

```txt
match
swipe
crush
```

## 22.4. Community

Use:

```txt
Community / Cộng đồng
CLB
```

Internal cần chọn convention:

```txt
community
club
```

Nếu CLB là subtype của community, nên model:

```txt
communities.type = club | faculty | class | study_group | topic
```

## 22.5. Audit

Use:

```txt
Audit log
Application audit log
SQL Server Audit
```

Do not merge them into one vague term:

```txt
log
```

Vì “log” có thể là debug log, access log, audit log, error log, hoặc một nơi chôn bí mật mà không ai đọc.
