# Agent Task Checklist

## 1. Before Starting

Agent must identify:

```txt
[ ] What module does this task belong to?
[ ] Is this product, API, DB, UI, security, test, or deployment work?
[ ] Which source-of-truth docs must be read?
[ ] Does this task affect MVP scope?
[ ] Does this task affect state machines?
[ ] Does this task affect API contract?
[ ] Does this task affect database schema?
[ ] Does this task affect privacy/security/moderation?
## 2. During Work

Agent must follow:

[ ] Use existing naming conventions.
[ ] Use English for code.
[ ] Use Vietnamese for UI copy.
[ ] Respect HCMUE-only user requirement.
[ ] Respect role/permission model.
[ ] Respect privacy settings.
[ ] Respect block/moderation rules.
[ ] Respect audit requirement for sensitive actions.
[ ] Respect notification/realtime side effects.
## 3. Before Finishing

Agent must verify:

[ ] Implementation matches feature spec.
[ ] API changes match api.yaml.
[ ] DB changes match schema/table specs.
[ ] UI changes match design system/page specs.
[ ] State transitions match state machine docs.
[ ] Error codes are documented.
[ ] Tests or acceptance criteria are identified.
[ ] Docs needing updates are listed.
[ ] Changelog update is needed or explicitly not needed.
## 4. Output Format

Every agent final output should include:

Summary
Files changed
Docs consulted
Docs updated
Tests needed
Risks / follow-up

# File 4 nên viết: `doc-sync-policy.md`

File này trả lời phần “tự động update docs” của bạn.

```md
# Documentation Sync Policy

## 1. Purpose

This file defines how UEConnect documentation stays synchronized with code, database, API, and design changes.

Docs are not optional.
Docs are part of the system.

---

## 2. Sync Strategy

UEConnect uses a hybrid documentation sync strategy:

```txt
Manual source-of-truth docs
+ Agent-assisted update
+ Pull request checklist
+ CI validation
+ Changelog discipline

The agent can propose and edit docs, but humans still review major business rules.

## 3. What Must Stay in Sync
| Change | Docs To Sync |
| --- | --- |
| New route/page | 03-product/sitemap.md, 04-design/page-specs/* |
| New feature | 03-product/feature-list.md, feature spec, acceptance criteria |
| New state | state machine docs, feature spec, API enum, DB enum |
| New API endpoint | 07-api/api.yaml, module API file, error codes |
| New DB table/column | 06-database/schema.md, table-specifications.md, erd.mmd |
| New permission | role-permission matrix, permission grants, API/admin docs |
| New admin action | admin spec, audit log rules, API docs |
| New notification | notification spec, API schema, browser push rules |
| New moderation behavior | moderation spec, safety spec, state machine |
| New design component | design system, component primitives/variants |
## 4. Agent Sync Workflow

When a code change happens, the agent must:

## 1. Detect impacted domain.
## 2. Read source-of-truth map.
## 3. Compare code change with docs.
4. Update impacted docs.
5. Update changelog if needed.
## 6. Add TODO only if information is truly missing.
## 7. Report docs updated in final summary.
## 5. CI Documentation Checks

Recommended CI checks:

lint markdown
validate api.yaml
validate mermaid diagrams
check broken relative links
check required frontmatter
check changelog updated for important folders

Suggested tools:

markdownlint
redocly lint
spectral
mermaid-cli
lychee or markdown-link-check
custom script for docs frontmatter
## 6. PR Checklist

Every pull request must answer:

[ ] Does this change affect product behavior?
[ ] Does this change affect API?
[ ] Does this change affect DB?
[ ] Does this change affect UI?
[ ] Does this change affect security/privacy?
[ ] Have related docs been updated?
[ ] Has api.yaml been updated if needed?
[ ] Has changelog been updated if needed?
## 7. Anti-patterns

Do not allow:

code changed but docs not updated
API changed but api.yaml stale
migration changed but ERD/schema stale
business rule hidden only in controller
state enum added only in code
admin action without audit docs
UI page added without sitemap/page-spec
