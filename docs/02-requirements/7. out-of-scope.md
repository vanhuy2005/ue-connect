# Out of Scope

## 1. Purpose

File này định nghĩa rõ các phần **không thuộc phạm vi** của UEConnect trong MVP và roadmap hiện tại.

Mục tiêu:

- Khóa scope để tránh scope creep.
- Làm rõ feature nào không làm trong MVP.
- Làm rõ feature nào có thể future-ready nhưng chưa implement.
- Làm rõ những hướng sản phẩm bị loại vì mâu thuẫn với vision.
- Giúp Product, Design, Engineering và QA biết khi nào cần từ chối yêu cầu mới.
- Bảo vệ định hướng UEConnect: HCMUE-only, verified-first, student-first, social nhưng không dating, permission-based, safety-first và governance-ready.

Out of scope không có nghĩa là feature đó “dở”.

Nó chỉ có nghĩa là:

```txt
Feature đó không phục vụ MVP hiện tại.
Feature đó làm lệch product positioning.
Feature đó tăng rủi ro trust/safety/privacy.
Feature đó làm scope quá lớn.
Feature đó cần policy/architecture riêng.
Feature đó chưa cần để chứng minh product value.
````

Một sản phẩm tốt không phải sản phẩm nhét mọi thứ vào. Đó gọi là bãi giữ xe feature, không phải product.

---

## 2. Scope Control Principles

UEConnect dùng các nguyên tắc kiểm soát scope sau:

```txt
01. Trust before growth
02. Student value before monetization
03. Safety before engagement
04. Verification before social access
05. Permission before admin power
06. Community governance before community freedom
07. Mentor support before career marketplace
08. PWA before native mobile app
09. HCMUE-only before multi-university expansion
10. Clarity before feature count
```

Nếu một feature mâu thuẫn với các nguyên tắc trên, feature đó phải bị loại khỏi MVP hoặc đưa vào future backlog có điều kiện.

# OOS-A. Dating / Matchmaking

## OOS-A001 — Dating app positioning

### Status

Rejected completely

### Description

UEConnect không phải dating app và không được định vị như dating app.

### Not Included

```txt
Dating profile
Romantic matching
Crush feature
Swipe-to-like
Hot profile
Relationship status discovery
Dating recommendation
Couple matching
Romantic intent filter
```

### Forbidden Terms

```txt
match
swipe
crush
hot
dating
hẹn hò
ghép đôi
tán tỉnh
người yêu
```

### Why Out of Scope

Dating positioning tạo rủi ro nghiêm trọng:

- Làm mất trust trong môi trường trường học.
- Làm Discovery bị hiểu sai.
- Làm nhà trường khó chấp nhận.
- Tăng nguy cơ harassment.
- Mâu thuẫn với product principle: social nhưng không dating.

### Allowed Alternative

Dùng:

```txt
Gửi lời chào
Kết nối
UEer
Bạn cùng khoa
Cùng học
Mentor
Community
CLB
```

### Related Business Rules

```txt
BR-A002
BR-G003
BR-Q003
```

# OOS-T. Technical Anti-scope

## OOS-T001 — Microservices architecture

### Status

Rejected for MVP

### Description

Không dùng microservices cho MVP.

### Why Out of Scope

UEConnect MVP phù hợp modular monolith với Laravel.

Microservices sẽ tăng:

- DevOps complexity.
- Deployment complexity.
- Distributed transaction.
- Debugging cost.
- Observability requirement.

Và rồi ta có 6 service để debug một nút “Gửi lời chào”. Một bi kịch công nghệ rất đắt tiền.

### Allowed Alternative

```txt
Laravel modular monolith
Domain services/actions
Clear module boundaries
Queue-ready design
```

---

# 5. Scope Decision Matrix

Khi có feature mới, đánh giá theo bảng này.

| Question                                                                    | If Yes             | If No            |
| --------------------------------------------------------------------------- | ------------------ | ---------------- |
| Feature có phục vụ verified HCMUE social value không?                       | Continue           | Reject / backlog |
| Feature có phục vụ student connection, mentor, community hoặc safety không? | Continue           | Reject / backlog |
| Feature có cần cho MVP metrics không?                                       | Consider P0/P1     | Likely P2/P3     |
| Feature có làm app giống dating không?                                      | Reject             | Continue         |
| Feature có làm app giống portal trường không?                               | Reconsider         | Continue         |
| Feature có làm scope thành super app không?                                 | Reject             | Continue         |
| Feature có cần legal/payment/complex integration không?                     | Future only        | Continue         |
| Feature có thể test và maintain trong MVP không?                            | Continue           | Backlog          |
| Feature có rủi ro privacy/safety chưa xử lý không?                          | Block until policy | Continue         |
| Feature có phù hợp PWA-first Laravel stack không?                           | Continue           | Reconsider       |

# 8. Out-of-scope Acceptance Criteria

A feature is confirmed out of scope if one or more conditions apply:

```txt
It conflicts with HCMUE-only direction.
It conflicts with verified-first model.
It makes Discovery feel like dating.
It requires payment/marketplace infrastructure.
It requires official academic system integration.
It requires native mobile app development.
It creates major privacy risk without policy.
It creates major moderation risk without tooling.
It expands product into a super app.
It does not support MVP success metrics.
It cannot be tested/maintained within MVP capacity.
```

---

# 9. Out-of-scope Review Checklist

Before adding a new feature, ask:

```txt
Does this feature help verified students connect meaningfully?
Does it support feed, discovery, mentor, community, safety or admin operation?
Does it map to business rules?
Does it map to a KPI?
Does it introduce new sensitive data?
Does it need moderation?
Does it need audit?
Does it need permission?
Does it make the app feel like dating?
Does it make the app feel like a portal?
Does it make the MVP harder to finish?
Can it be delayed without harming MVP value?
```

Nếu câu trả lời cuối là “có thể delay”, thì delay. MVP không cần chứng minh rằng con người có thể phát minh vô hạn nút bấm.
